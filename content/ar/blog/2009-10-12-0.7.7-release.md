يتضمن إصدار I2P 0.7.7 عدة تحسينات على إخفاء الهوية. أُعيدت هيكلة استخدام مفاتيح الجلسة لأغراض التشفير بالكامل، وأصبح I2PSnark الآن يستخدم tunnels الخاصة به للتواصل مع الـ tracker. ويتواصل العمل على تحسين لوحة تحكم الـ router، بما في ذلك الامتثال لمعايير HTML وتحسين دعم UTF-8.

يستمر العمل أيضًا على زيادة السرعات، وذلك بتقليل عدد مفاتيح الجلسة المنقولة، وتحسين آليات إسقاط الرسائل أثناء حالات الحمل الزائد، وتقليل عمليات الإسقاط من قِبل routers ذات عرض نطاق ترددي عالٍ. يوجد خيار تحديث جديد غير موقّع لمن يرغب في الحصول على تحديثات تلقائية إلى الإصدارات التطويرية الأحدث للغاية.

يتضمن هذا الإصدار أيضًا عدة تغييرات لتقليل استهلاك الذاكرة ووحدة المعالجة المركزية، ومجموعة كبيرة من إصلاحات الأخطاء. يُوصى بالترقية. يُرجى تزويد المطورين بملاحظاتكم عبر IRC #i2p أو [forum.i2p2.de](http://forum.i2p2.de) و[شارك](http://www.i2p2.de/getinvolved.html)!

## تفاصيل الإصدار:

### Resources

- Redesign bloom filters, saves 6-8MB heap.
- Reduce PRNG buffers for I2PAppContext, saves 3.5MB heap.
- Remove more unused classes and code, more Java 5 cleanups
- Reduce connection limits slightly.

### الموارد

- Improve the random early discarder, to be smarter - will hopefully improve tunnel speeds
- Increase the size of the Bloom filter for high bandwidth routers (>= 512KBps share) to reduce packet discards

### Tunnels

- Use multiple Session Key Managers for anonymity benefits
- Reduce the number of Session Tags sent out at once, for efficiency - should reduce stalls on long-lived connections

### التشفير

- Lots of HTML cleanups and fixes
- UTF-8 fixes
- New error page

### وحدة التحكم

- New unsigned update option
- Verify zip file integrity at startup

### التحديثات

- Use snark's tunnels rather than the HTTP Proxy for announces and torrent fetches, for efficiency and anonymity.
