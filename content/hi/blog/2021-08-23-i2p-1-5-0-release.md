## अद्यतन विवरण

हाँ, बिल्कुल सही, 0.9.x रिलीज़ के 9 वर्षों के बाद, हम सीधे 0.9.50 से 1.5.0 पर जा रहे हैं। यह किसी बड़े API परिवर्तन का संकेत नहीं है, और न ही यह दावा है कि विकास अब पूरा हो गया है। यह केवल हमारे उपयोगकर्ताओं को गुमनामी और सुरक्षा प्रदान करने के लिए लगभग 20 वर्षों के कार्य की मान्यता है।

यह रिलीज़ बैंडविड्थ कम करने के लिए छोटे tunnel build संदेशों के कार्यान्वयन को पूरा करती है। हम नेटवर्क के routers को X25519 encryption पर स्थानांतरित करने की प्रक्रिया जारी रखते हैं। स्वाभाविक रूप से, कई बग फिक्स और प्रदर्शन में सुधार भी शामिल हैं।

हमेशा की तरह, हम अनुशंसा करते हैं कि आप इस रिलीज़ में अपडेट करें। सुरक्षा बनाए रखने और नेटवर्क की मदद करने का सबसे अच्छा तरीका नवीनतम रिलीज़ चलाना है।

## रिलीज़ विवरण

**परिवर्तन**

- RRD4J 3.8
- Tunnels: Finish support for new build messages (proposal 157)
- Updates: Support for .dmg and .exe updates

**त्रुटि सुधार**

- Console: Fix generation of SSL keys on Java 17
- i2psnark: Fix autostart for magnets
- Router: Fix rare deadlock in publishing our RI
- SSU: Fix handling of bad peer test responses
- UPnP: IPv6 fixes

**अन्य**

- Jetty: Improve sort in directory listings
- Jetty: Add X-I2P-Location header
- Router: Increase probability to rekey to ECIES
- Streaming: Performance improvements for low-latency connections
- Translation updates
