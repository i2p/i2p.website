I2P संस्करण 0.7.7 में अनामिता में कई सुधार शामिल हैं। क्रिप्टोग्राफी के लिए session keys (सेशन कुंजियाँ) के उपयोग को पूरी तरह से पुनःडिज़ाइन किया गया, और I2PSnark अब ट्रैकर संचार के लिए अपने स्वयं के tunnels का उपयोग करता है। router कंसोल को बेहतर बनाने का कार्य जारी है, जिसमें HTML मानकों के अनुपालन और UTF-8 के बेहतर समर्थन शामिल हैं।

गति बढ़ाने पर भी काम जारी है, जिसमें स्थानांतरित की जाने वाली सेशन कुंजियों की संख्या कम करना, ओवरलोड के दौरान संदेशों को ड्रॉप (त्याग) करने की विधियों में सुधार करना, और उच्च-बैंडविड्थ routers द्वारा होने वाले ड्रॉप्स को कम करना शामिल है। जो लोग नवीनतम (bleeding-edge) विकास संस्करणों के लिए स्वचालित अपडेट चाहते हैं, उनके लिए एक नया अनहस्ताक्षरित अपडेट विकल्प उपलब्ध है।

इस रिलीज़ में मेमोरी और CPU उपयोग को कम करने के लिए कई बदलाव, और बग फिक्स का बड़ा संग्रह भी शामिल हैं। अपग्रेड करने की अनुशंसा की जाती है। कृपया डेवलपर्स को IRC #i2p या [forum.i2p2.de](http://forum.i2p2.de) पर फीडबैक दें, और [शामिल हों](http://www.i2p2.de/getinvolved.html)!

## रिलीज़ विवरण:

### Resources

- Redesign bloom filters, saves 6-8MB heap.
- Reduce PRNG buffers for I2PAppContext, saves 3.5MB heap.
- Remove more unused classes and code, more Java 5 cleanups
- Reduce connection limits slightly.

### संसाधन

- Improve the random early discarder, to be smarter - will hopefully improve tunnel speeds
- Increase the size of the Bloom filter for high bandwidth routers (>= 512KBps share) to reduce packet discards

### Tunnels

- Use multiple Session Key Managers for anonymity benefits
- Reduce the number of Session Tags sent out at once, for efficiency - should reduce stalls on long-lived connections

### क्रिप्टोग्राफी

- Lots of HTML cleanups and fixes
- UTF-8 fixes
- New error page

### कंसोल

- New unsigned update option
- Verify zip file integrity at startup

### अद्यतन

- Use snark's tunnels rather than the HTTP Proxy for announces and torrent fetches, for efficiency and anonymity.
