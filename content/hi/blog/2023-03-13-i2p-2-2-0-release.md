हमने 2.2.0 रिलीज़ की तारीख आगे कर दी है, जो आज, 13 मार्च, 2023 को होगी। इस रिलीज़ में NetDB, Floodfill, और Peer-Selection (पीयर-चयन) कॉम्पोनेंट्स में बदलाव शामिल हैं, जो router की DDOS हमलों से बचे रहने की क्षमता में सुधार करते हैं। हमले संभवतः जारी रहेंगे, लेकिन इन प्रणालियों में किए गए सुधार DDOS हमलों के जोखिम को कम करने में मदद करेंगे, क्योंकि ये router को दुर्भावनापूर्ण प्रतीत होने वाले routers की पहचान करने और उन्हें कम प्राथमिकता देने में सहायता करेंगे।

यह रिलीज़ स्ट्रीमिंग उपप्रणाली में रीप्ले सुरक्षा भी जोड़ती है, जो ऐसे हमलावर को रोकती है जो किसी एन्क्रिप्टेड पैकेट को कैप्चर कर सकता है, ताकि वह उसे अनपेक्षित प्राप्तकर्ताओं को भेजकर उसका पुन: उपयोग न कर सके। यह परिवर्तन पिछले संस्करणों के साथ संगत है, इसलिए पुराने routers नए routers की स्ट्रीमिंग क्षमताओं का उपयोग कर सकेंगे। यह समस्या I2P विकास टीम द्वारा आंतरिक रूप से खोजी और ठीक की गई थी, और इसका DDOS हमलों से कोई संबंध नहीं है। हमें कभी भी वास्तविक दुनिया में रीप्ले किया गया स्ट्रीमिंग पैकेट नहीं मिला है, और हमें नहीं लगता कि अब तक I2P नेटवर्क के खिलाफ कभी भी स्ट्रीमिंग रीप्ले हमला हुआ है।

जैसा कि आपने ध्यान दिया होगा, इन रिलीज़ नोट्स और स्वयं रिलीज़ पर idk के हस्ताक्षर हैं, zzz के नहीं। zzz ने परियोजना से अलग होने का निर्णय लिया है और उनकी ज़िम्मेदारियाँ अब अन्य टीम सदस्यों द्वारा संभाली जा रही हैं। इसी कारण, परियोजना नेटवर्क सांख्यिकी अवसंरचना को प्रतिस्थापित करने और डेवलपमेंट फ़ोरम को i2pforum.i2p पर स्थानांतरित करने पर काम कर रही है। हम zzz को इतने लंबे समय तक ये सेवाएँ प्रदान करने के लिए धन्यवाद देते हैं।

हमेशा की तरह, हम अनुशंसा करते हैं कि आप इस रिलीज़ पर अपडेट करें। सुरक्षा बनाए रखने और नेटवर्क की मदद करने का सबसे अच्छा तरीका नवीनतम रिलीज़ चलाना है।

## विवरण

### Changes

- i2psnark: New search feature
- i2psnark: New max files per torrent config
- NetDB: Expiration improvements
- NetDB: More restrictions on lookups and exploration
- NetDB: Store handling improvements
- NTCP2: Banning improvements
- Profiles: Adjust capacity estimates
- Profiles: Expiration improvements
- Router: Initial support for congestion caps (proposal 162)
- Transports: Add inbound connection limiting
- Tunnels: Refactor and improve peer selection
- Tunnels: Improve handling of "probabalistic" rejections
- Tunnels: Reduce usage of unreachable and floodfill routers

### परिवर्तन

- Docker: Fix graphs not displaying
- i2psnark: Fix torrents with '#' in the name
- i2psnark standalone: Fix running from outside directory
- i2psnark standalone: Remove "Start I2P" menu item from systray
- i2ptunnel: Fix typo in HTTPS outproxy hostname
- i2ptunnel: Interrupt tunnel build if stop button clicked
- i2ptunnel: Return error message to IRC, HTTP, and SOCKS clients on failure to build tunnels
- NTCP2: Ensure an IPv6 address is published when firewalled and IPv4 is not
- Ratchet: Don't bundle wrong leaseset with ack
- Router: Fixes for symmetric NAT errors on 'full cone' NAT
- SAM: Interrupt tunnel build if client times out
- SSU2: Fix rare peer test NPE
- Sybil: Don't blame i2pd publishing ::1
- Sybil: Memory usage and priority reduction
- Transports: More IP checks

### त्रुटि सुधार

- Blocklist efficiency improvements
- Bundles: Identify Win and Mac bundles in version info
- Console: Identify service installs, revision, and build time in version info
- Console: NetDB search form and tunnels page improvements (advanced only)
- Router: Reduce stats memory usage
- Tunnels: Reduce "grace period"
- Translation updates
