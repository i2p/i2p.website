0.8.3 संस्करण में थ्रेड्स और मेमोरी उपयोग में कमी तथा तेज़ I2CP (client-router) संचार सहित कई प्रदर्शन सुधार शामिल हैं।

अब नया SSL router console समर्थन भी है, एक नया reseed configuration पेज जिसमें प्रतिबंधात्मक फायरवॉल के पीछे वालों के लिए HTTP proxy समर्थन शामिल है, remote clients के लिए I2CP-over-SSL समर्थन सहित एक नया I2CP configuration अनुभाग, बेहतर DoS (सेवा-अस्वीकार) सुरक्षा के लिए एक नया server connection limits और blacklist configuration अनुभाग, और वैकल्पिक jump servers आसानी से जोड़ने के लिए एक नया HTTP proxy jump server configuration अनुभाग। मेमोरी उपयोग कम करने के लिए आँकड़े अब डिफ़ॉल्ट रूप से सीमित हैं; आँकड़ों का पूरा सेट stats configuration पेज पर फिर से सक्षम किया जा सकता है। बेशक, कुछ bug fixes भी हैं, इसलिए हमेशा की तरह, अपग्रेड करने की सिफारिश की जाती है।

कृपया नेटवर्क को बढ़ाने में मदद करें। [शामिल हों](http://www.i2p2.de/getinvolved.html), जानकारी फैलाएँ, और [दान करें](http://www.i2p2.de/donate.html)! यदि आपको कोई बग मिले, तो कृपया [gitlab](http://i2pgit.org/i2p-hackers/i2p.i2p/issues) पर एक रिपोर्ट दर्ज करें। हम नए और मौजूदा अनुवादों के लिए अभी भी सहायता की तलाश में हैं। कृपया IRC #i2p पर स्वयंसेवक के रूप में जुड़ें।

फ़ाइलें [डाउनलोड पृष्ठ](/downloads/) पर उपलब्ध हैं।

**रिलीज़ विवरण:**

## नई विशेषताएँ

* SSL Console
* SSL I2CP
* Jump list, blacklist, and connection limits now on i2ptunnel edit forms
* Smarter update handler that tries multiple sources
* New reseed configuration page for those behind restrictive firewalls
* New I2CP configuration form

## संसाधन में कमी

* More efficient I2CP for internal clients
* Less threads, better thread pooling
* Adjust thread count based on available memory
* Change full stats default to false
* Caching of several data structures
* Naming lookup support over existing I2CP sessions

## त्रुटि सुधार

* Fix i2psnark buttons in Firefox 4.0b and IE
* Restore library functions required by Syndie
