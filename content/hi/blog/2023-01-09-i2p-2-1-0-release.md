## अद्यतन विवरण

नवंबर में हमारे 2.0.0 रिलीज़ के बाद से हमने कई बातें सीखी हैं। जैसे-जैसे routers उस रिलीज़ पर अपडेट हुए हैं, नेटवर्क में हमारे नए SSU2 ट्रांसपोर्ट प्रोटोकॉल के समर्थन का स्तर लगभग 1% से बढ़कर 60% से अधिक हो गया है। पहला, हमने यह सुनिश्चित किया है कि SSU2 एक मज़बूत, अच्छी तरह से डिज़ाइन किया गया और सुरक्षित प्रोटोकॉल है। दूसरा, हालांकि, हमने प्रोटोकॉल के कार्यान्वयन में कई छोटे या कभी-कभार सक्रिय होने वाले बग्स ढूंढे और ठीक किए हैं। कुल मिलाकर, इन बग्स के प्रभाव ने नेटवर्क के प्रदर्शन को कम कर दिया है।

साथ ही, हम नेटवर्क में बढ़ी हुई tunnel की संख्या और घटती tunnel build सफलता दर से अवगत हैं, जो संभवतः Bitcoin की नई I2P transient address (अस्थायी address) सुविधा से ट्रिगर हुए हैं, लेकिन हमारे SSU2 बग्स और अन्य congestion control (ट्रैफिक भीड़ नियंत्रण) समस्याओं के कारण स्थिति और बिगड़ गई है। हम I2P नेटवर्क पर मांग को कम करने के लिए Bitcoin और अन्य गैर-Bitcoin प्रोजेक्ट्स के साथ काम कर रहे हैं। हमने भीड़भाड़ के समय नेटवर्क पर भार कम करने के लिए अपने एल्गोरिदम में सुधार किया है। हम i2pd के साथ भी सामान्य congestion control रणनीतियाँ विकसित करने के लिए सहयोग कर रहे हैं।

इसलिए, हमने सुधार सभी तक पहुँचाने के लिए इस रिलीज़ को लगभग छह सप्ताह पहले ला दिया है। i2pd ने पिछले सप्ताह अपना संस्करण 2.45.0 जारी किया, और शुरुआती परिणाम उत्साहजनक हैं। नए प्रोटोकॉल और वितरित नेटवर्क विकसित करना कठिन होता है। भीड़भाड़ अक्सर बहुत कम चेतावनी के साथ आ सकती है और कारण के बारे में बहुत कम संकेत देती है। समस्याओं का निदान करने और, उम्मीद है, उन्हें ठीक करने के दौरान आपके धैर्य के लिए धन्यवाद।

हमेशा की तरह, हम अनुशंसा करते हैं कि आप इस रिलीज़ पर अद्यतन करें। सुरक्षा बनाए रखने और नेटवर्क की सहायता करने का सर्वोत्तम तरीका नवीनतम रिलीज़ चलाना है।

## रिलीज़ विवरण

### Changes

- Console: New status and banned peers tabs on /peers
- i2ptunnel: Add torsocks support
- i2ptunnel: Add SOCKS tunnel conversion to CONNECT outproxy
- i2ptunnel: Add SOCKS outproxy port configuration
- i2ptunnel: Update encryption type defaults
- Router: Improved congestion detection and handling
- Router: Use compressible padding for destinations and router infos (proposal 161)
- SSU: Redesign symmetric NAT detection

### Bug Fixes

- Console: Fix configuration for Argentinian Spanish
- Crypto: Fix LS2 encrypted leasesets, broken since 1.8.0
- i2psnark: Avoid OOM starting large number of torrents
- i2ptunnel: Numerous SOCKS tunnel fixes
- NTCP: Fix rare termination NPE
- Profiles: Fix profile load stopping after hitting corrupt file
- Router: Clock skew handling fixes and improvements
- SSU: Don't publish IPv4 address when configured for IPv6-only
- SSU: Fix handling of banned peers
- SSU2: Peer Test fixes and improvements
- SSU2: Termination fixes and improvements
- SSU2: Token and handshake fixes and improvements
- SSU2: Fix rare packet handling NPE
- SSU2: Fix rare termination IAE
- SSU2: Fix retransmission of session confirmed
- SSU2: Fix attempted connection to ourselves as an introducer
- UPnP: Catch rare assertion error

### परिवर्तन

- Console: Add leaseset lookup to advanced search form
- i2psnark: Add partial Dutch translation
- i2ptunnel: Allow IRCv3 ACCOUNT and CHGHOST through filter
- SSU2: Preliminary support for disabling SSU1
- Sybil: Add IPv6 address tests
- Translation updates
