A versão 0.7.11 corrige vários problemas, alguns relacionados a relógios, que impediam que novos routers se integrassem rapidamente à rede. Também corrige o instalador para Windows de 64 bits. Esta versão inclui uma atualização para o [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) para corrigir [esta vulnerabilidade](http://www.kb.cert.org/vuls/id/402580).

Como de costume, a versão inclui mais melhorias de anonimato e desempenho, além de uma variedade de correções de bugs. Recomenda-se a atualização.

Por favor, envie feedback aos desenvolvedores no IRC #i2p ou no [forum.i2p2.de](http://forum.i2p2.de/), [envolva-se](http://www.i2p2.de/getinvolved.html), ajude a divulgar e [doe](http://www.i2p2.de/donate.html)! Ainda estamos à procura de ajuda com traduções novas e existentes. Voluntarie-se no IRC #i2p.

Os arquivos estão disponíveis na [página de download](/downloads/).

Detalhes da versão:

## Instalador

- Fix installations on 64-bit Windows by upgrading to launch4j 3.0.1 and adding 64-bit dll to izpack.
- The wrapper is still 32-bit only. 64-bit Windows users should start I2P with the 'No Window' icon if the 'Restartable' icon does not work.

## Transporte

- Better handle clock skews at startup, even if NTP does not work because UDP is blocked or other reasons
- Fix clock skew reporting
- Bootstrap profiles when loading the netDb from disk (broken in 0.7.8)

## Servidor web incorporado (Jetty)

- Upgrade to [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) which fixes [this vulnerability](http://www.kb.cert.org/vuls/id/402580).
- Turn on checkAliases for increased protection

## Banco de Dados da Rede

- Add some DOS protection for floodfills
- Increase the number of floodfills again

## Outros

- Faster startup
- Bug fixes in i2psnark, BOB, clock handling, and more
