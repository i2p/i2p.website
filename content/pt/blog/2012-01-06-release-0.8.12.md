A versão 0.8.12 corrige vários erros que causavam corrupção de mensagens, alguns presentes desde 2005. Ela também inclui um redesenho do controle de congestionamento do router e a otimização contínua do uso de CPU e memória. Esperamos que essas mudanças melhorem o desempenho da rede. Recomenda-se a atualização.

Os arquivos estão disponíveis na [página de download](/downloads).

## DETALHES DA VERSÃO

Esta versão inclui várias mudanças significativas que devem ajudar a melhorar a confiabilidade e o desempenho da rede.

As duas primeiras alterações são correções para bugs graves na transmissão de pacotes UDP, descritos abaixo. A terceira alteração consiste em correções no descompressor. Corrigindo-as deve eliminar uma grande fonte de falhas na entrega de mensagens.

Além disso, esta versão inclui uma reescrita do controle de congestionamento do tráfego participante do router. Agora ele deve medir com mais precisão a largura de banda participante atual, lidar melhor com picos, descartar menos mensagens e, se for necessário descartar, descartar as mensagens corretas.

Além disso, esta versão reduz a quantidade de processamento que os routers realizam para as mensagens que encaminham pelo tunnel. Essas mensagens não precisam ser completamente analisadas sintaticamente e validadas, pois o gateway está simplesmente encaminhando-as. Isso ajudará o desempenho de todos os routers, mas a melhoria provavelmente não será perceptível, exceto em routers de alto tráfego.

Estamos otimistas de que essas mudanças ajudarão e, claro, eliminar as fontes de corrupção de mensagens certamente ajudará. No entanto, a extensão da melhoria não será evidente até que a maioria da rede seja atualizada.

Esta versão também contém algumas atualizações no tema claro do console do router. Pode ser necessário fazer um shift-reload ou control-reload no seu navegador para forçar o recarregamento do CSS.

## Principais mudanças

- Instead of fully parsing and validating messages received at the inbound gateway, simply pass them down the tunnel
- Don't verify I2NP checksums in most cases, as message corruption is caught at other layers
- Don't recalculate I2NP checksums on messages passed through unchanged
- Several NTCP pumper optimizations, to hopefully fix NTCP pumper high CPU usage on fast routers
- Rewrite participating tunnel congestion control, to drop less, more accurately measure traffic, and handle bursts better

## Atualização do Wrapper

Novas instalações incluem a versão 3.5.13 do wrapper, que corrige uma corrupção do heap causada por linhas de log muito longas. Consulte http://wrapper.tanukisoftware.com/doc/english/release-notes.html para informações adicionais. Usuários do pacote PPA do I2P devem ter recebido esta atualização na última semana de dezembro. Se você não usa nosso pacote PPA e gostaria de atualizar seu wrapper manualmente, siga as instruções em http://www.i2p2.de/manualwrapper.

## Correções de bugs

- Fix major SSU MTU bug introduced in 0.8.9. The router sometimes sent UDP packets that exceeded the maximum size that routers would accept on the receive side. This resulted in a lot of dropped packets, tunnel build problems, and connection problems. One thing that contributed to us missing the problem was that 0.8.9 was released in the middle of the huge network expansion in early October, when network performance was already deteriorating rapidly.
- Fix major SSU fragmentation bug present since 2005. UDP corrupted transmit messages when the message size was an exact multiple of the fragmentation size.
- Fix major decompression bugs present since 2005 that caused message corruption and data loss at multiple protocol layers.
- Snark doesn't always delete directories
- Fix all character case conversion (Turkish four i problem)
- Whitelist more IRC commands
- Remove expired reseed cert

## Outros

- Speed up the inefficient GeoIP lookup
- Optimize several heavily-used utility functions
- Don't drop the transport connection when receiving an invalid message, it isn't necessary and could be an attack vector
- Console light theme update
- Move the complex network configuration stuff to a new console page
- Add a link to hide the news in the console
- Allow numerous additional IRC commands through the IRC filter
- Several other cleanups, optimizations, and object caches added
- New Estonian translation (thanks ajutine)
- Spanish, Swedish, Ukrainian translation updates
