A versão 0.9.7 inclui correções de bugs significativas e melhorias significativas.

Pela primeira vez, routers da classe 'N' (aqueles com um mínimo de 128 KBytes/sec de largura de banda compartilhada) passarão automaticamente a se tornar floodfill (antes, eram apenas routers 'O' com 256 KBps). Isso aumentará a população de floodfill para resistência adicional a certos ataques (veja abaixo). Routers floodfill não consomem muita largura de banda adicional, mas tendem a usar memória adicional e mais conexões simultâneas. Se você não deseja que seu router se torne floodfill, defina a configuração avançada router.floodfillParticipant=false.

Como acreditamos que a última versão corrigiu os erros experimentais de atualização via torrent, 3% dos routers devem atualizar via BitTorrent na própria rede neste ciclo.

As verificações de atualização de plugins, possivelmente com falhas em várias versões, foram corrigidas. Seus plugins devem voltar a se atualizar automaticamente após atualizar o router.

Corrigimos um bug grave no temporizador de streaming que contribuía para desconexões frequentes no IRC.

Esta versão contém mitigações adicionais em resposta ao [artigo sobre "ataques práticos"](http://wwwcip.informatik.uni-erlangen.de/~spjsschl/i2p.pdf). No entanto, ainda temos muito trabalho a fazer para resistir a ataques Sybil nos nós floodfill e resistir à análise de tráfego nos gateways e nos pontos finais dos tunnels exploratórios. É um bom lembrete para todos de que nossa rede ainda é relativamente pequena e vulnerável. Atualmente, não recomendamos quaisquer usos que coloquem alguém em grave perigo. Continuaremos trabalhando para melhorá-la... por favor, continuem ajudando a divulgá-la. Uma rede maior é uma rede melhor.

## DETALHES DO LANÇAMENTO

### Anonymity Improvements

- End-to-end encryption of responses to leaseset lookups
- Expand floodfill pool by enabling class 'N' floodfills
- Randomize padding inside encrypted SSU packets
- Preparation for better SSU protocol obfuscation

### Melhorias no anonimato

- Fix newer lease sets not getting stored or published
- Fix classpath bug when used with 4-year-old installations, causing the console not to start
- Fix addressbook database bug preventing update of the reverse index
- Fix i2psnark bug that changed the infohash of torrents created by Robert and fetched via magnet link
- Fix version checking for plugins
- Fix a streaming timer bug causing frequent IRC disconnects (also affects other close-on-idle tunnels)

### Correções de erros

- Don't install as a service on Windows by default
- Reduce transport idle timeouts
- Reduce tunnels on idle in i2psnark
- Change default in i2ptunnel GUI to 3 hops
- IE 10 support
- Individual expiration times in leases, for efficiency on destinations with a high number of tunnels
- Low-level encryption and XOR speedups
- Jetty 7.6.11
- Tomcat 6.0.37
- Translation updates: Chinese, French, German, Portuguese, Russian, Spanish
- New Turkish translation
- Wrapper 3.5.19 (new installs and PPA only)
- Update GeoIP data (new installs and PPA only)
