I2P agora requer Java 6 ou superior. Recomendamos fortemente que você atualize para o Java 7. Se você ainda estiver usando o Java 5, deve atualizar o Java antes de instalar o I2P 0.9.12.

A versão 0.9.12 adiciona suporte preliminar a Destinos assinados com ECDSA. Ela contém várias correções no tratamento de Mensagens de Status de Entrega (confirmações de recebimento) e essas mensagens agora são criptografadas de ponta a ponta para maior segurança.

Atualizamos para o Jetty 8. O Jetty 8 é quase idêntico ao Jetty 7, portanto não há conversões complexas de arquivos de configuração como houve em atualizações anteriores do Jetty. Nenhuma alteração manual deve ser necessária.

Como de costume, recomendamos que você atualize para esta versão. A melhor maneira de manter a segurança e ajudar a rede é executar a versão mais recente.

No início de março, a empresa de busca na Internet [DuckDuckGo](https://duckduckgo.com/) [doou](https://duck.co/blog/foss2014) US$ 5.000 ao Invisible Internet Project (I2P) como parte de seu programa anual de doações a projetos de código aberto. O prêmio foi concedido com base em [indicações](https://duck.co/forum/thread/5174/foss-donation-nomations-2014-edition) feitas por membros do público no portal da comunidade do DuckDuckGo.

Isto marca a maior doação individual já recebida pelo I2P e reflete um interesse crescente em privacidade e segurança por parte da comunidade da Internet. O financiamento ajudará o I2P a alcançar mais usuários, ampliar o desenvolvimento e auditar o código. Também permitirá que os desenvolvedores do I2P participem de conferências, como a [Real-World Cryptography](http://www.realworldcrypto.com/) conferência na cidade de Nova York, onde os desenvolvedores se reuniram e colaboraram com especialistas em criptografia, em consonância com os objetivos do I2P de fornecer anonimato ao público.

O I2P agradece a Gabriel Weinberg e ao DuckDuckGo pela generosa doação e à comunidade I2P pelo apoio nas indicações.

## DETALHES DA VERSÃO

### Anonymity Improvements

- Encrypt Delivery Status Messages
- Add preliminary support for ECDSA-signed Destinations
- Add check for replayed NTCP session requests
- Add throttling and blocking checks to streaming ping processing

### Melhorias de Anonimato

- Fix RouterInfo exchange in NTCP
- Extend timeout for Delivery Status Messages
- Drop streaming messages from recently closed connections
- Fix restarts on Raspberry Pi
- Restore profileOrganizer.sameCountryBonus advanced config
- Fix for jwebcache and i2phex

### Correções de erros

- Jetty 8.1.14.v20131031; Java 6 now required
- Reduce target connection count again to reduce tunnel reject rate further
- Add rate limit for outbound connections at tunnel endpoints
- Add optional inproxy blocking in i2ptunnel
- Use SSU session key for relay request/response when available
- Include HTTP POST data in SYN packet
- Add getopt library for better argument processing
- More removal of Jetty dependencies
- Remove MD5 code, use Java libraries instead
- Change the default addressbook subscription URL
- Translation updates
- Update GeoIP data (new installs and PPA only)
