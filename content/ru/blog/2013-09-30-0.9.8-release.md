0.9.8 включает долгожданную поддержку IPv6. Она включена по умолчанию, но для использования вам нужен публичный IPv6-адрес. Настройка осуществляется на вкладке конфигурации 'network' в вашей консоли. У нас также есть улучшения анонимности, включая добавление заполнения (padding) для пакетов SSU и более длинные закрытые ключи router.

30% из вас обновятся через внутрисетевой торрент в этом цикле обновления.

## Подробности об IPv6

IPv6 по умолчанию включён и имеет приоритет. Если у вас есть публичный IPv6-адрес и вы подключаетесь к другому router с опубликованным IPv6-адресом, соединение будет установлено по IPv6. В консоли router на странице /confignet появилась новая секция настройки IPv6. Если IPv6 вызывает проблемы, вы можете отключить его там.

В рамках работ по развитию IPv6 I2P теперь поддерживает публикацию нескольких IP-адресов. Если у вас несколько публичных IP-адресов (IPv4, IPv6 или оба варианта), вы можете включать или отключать их по отдельности на /confignet. По умолчанию используются первые обнаруженные адреса IPv4 и IPv6. Если у вас несколько адресов, вам следует проверить конфигурацию на /confignet и при необходимости скорректировать ее. Обратите внимание: хотя на /confignet можно включить несколько адресов IPv4 и IPv6, мы рекомендуем использовать только по одному адресу IPv4 и IPv6. При работе с несколькими адресами каждого типа все еще остаются ошибки, которые предстоит исправить.

Хотя поддержка IPv6 проектировалась и разрабатывалась в течение нескольких лет, она была протестирована лишь ограниченным числом пользователей и всё ещё находится в статусе бета. Если у вас есть публичный IPv6-адрес, пожалуйста, следите за своим router и журналами на предмет проблем и при необходимости отключайте его. Пожалуйста, сообщайте о найденных ошибках на http://trac.i2p2.i2p.

## Подробности смены ключей

Для тех из вас, кто запускает I2P на более производительном оборудовании (как правило, 64‑битной x86), router сгенерирует новую идентичность с использованием более длинных ключей. Это существенно снизит ваш ретрансляционный трафик на 48 часов и более, пока ваш router вновь интегрируется в сеть. Из-за новых ключей, большого числа обновлений торрентов и недавнего роста сети мы ожидаем существенные перебои в работе сети в течение недели или дольше после выпуска обновления. Пожалуйста, проявите терпение, ситуация должна начать улучшаться через несколько дней.

Эти изменения могут привести к более высокой загрузке процессора у некоторых из вас. Мы делаем всё возможное, чтобы повысить эффективность, но более высокий уровень безопасности обычно требует больше вычислений. Производительность также может быть низкой в течение первой недели из-за network churn (частой смены узлов в сети). Мы оценим производительность сети, прежде чем решать, менять ли длину ключа на более медленном оборудовании в одном из будущих релизов.

За последние несколько недель мы наблюдаем стремительный рост сети, что для некоторых оборачивается небольшими сложностями, особенно по выходным. Однако сеть по-прежнему работает достаточно хорошо, так что продолжайте распространять информацию.

## Грядут новые изменения

Мы на начальном этапе разработки крупных изменений для усиления нашей криптографии. Более стойкая криптография будет использовать больше ресурсов процессора и, возможно, потребует как минимум Java 7 JRE. Мы понимаем ваше желание запускать I2P на маломощном и/или старом оборудовании. Мы усердно работаем над тем, чтобы минимизировать влияние, но некоторое снижение производительности неизбежно. Кроме того, Java 5 и 6 больше не поддерживаются компанией Oracle. Сейчас самое время обновиться до Java 7. О любых изменениях минимальных требований будет объявлено заблаговременно.

## СВЕДЕНИЯ О ВЫПУСКЕ

### Major Changes

- IPv6 support for both NTCP and SSU

### Anonymity Improvements

- SSU protocol obfuscation by adding random padding
- Longer encryption and DH private keys for users on faster platforms

### Bug Fixes

- Fix I2PTunnel / I2CP locking and duplicates (partial)
- Fix translation of HTTP proxy error pages
- Fix occasional runtime exception in NTCP

### Other

- Big rework of transport code to accommodate multiple addresses and IPv6
- Streaming: Improved recovery from lost acks, other fixes
- Use Transifex for translation of initial news and HTTP proxy error pages
- Translation updates: Chinese, French, German, Portuguese, Russian, Swedish, Turkish
- New Romanian translation
- Jetty 7.6.12.v20130726
- Wrapper 3.5.20 (new installs and PPA only)
- Update GeoIP data (new installs and PPA only)
