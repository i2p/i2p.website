Версия I2P 0.7.7 содержит несколько улучшений анонимности. Использование сеансовых ключей для криптографии было полностью переработано, и I2PSnark теперь использует собственные tunnels для взаимодействия с трекером. Работа по улучшению консоли router продолжается, включая соответствие стандартам HTML и улучшенную поддержку UTF-8.

Работа также продолжается над увеличением скоростей — как за счет уменьшения числа передаваемых сеансовых ключей, так и за счет улучшения методов отбрасывания сообщений при перегрузке, а также за счет уменьшения числа отбрасываний на routers с высокой пропускной способностью. Появилась новая опция неподписанного обновления для тех, кто хотел бы получать автоматические обновления до самых передовых экспериментальных версий.

В этом выпуске также есть несколько изменений для снижения потребления памяти и ЦП, а также множество исправлений ошибок. Рекомендуется обновиться. Пожалуйста, оставьте разработчикам отзыв в IRC #i2p или на [forum.i2p2.de](http://forum.i2p2.de) и [примите участие](http://www.i2p2.de/getinvolved.html)!

## Сведения о выпуске:

### Resources

- Redesign bloom filters, saves 6-8MB heap.
- Reduce PRNG buffers for I2PAppContext, saves 3.5MB heap.
- Remove more unused classes and code, more Java 5 cleanups
- Reduce connection limits slightly.

### Ресурсы

- Improve the random early discarder, to be smarter - will hopefully improve tunnel speeds
- Increase the size of the Bloom filter for high bandwidth routers (>= 512KBps share) to reduce packet discards

### Tunnels

- Use multiple Session Key Managers for anonymity benefits
- Reduce the number of Session Tags sent out at once, for efficiency - should reduce stalls on long-lived connections

### Криптография

- Lots of HTML cleanups and fixes
- UTF-8 fixes
- New error page

### Консоль

- New unsigned update option
- Verify zip file integrity at startup

### Обновления

- Use snark's tunnels rather than the HTTP Proxy for announces and torrent fetches, for efficiency and anonymity.
