Релиз 0.7.11 исправляет несколько проблем, в том числе связанные с системным временем, которые мешали новым routers быстро интегрироваться в сеть. Он также исправляет установщик для 64-битной Windows. Релиз включает обновление до [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) для устранения [этой уязвимости](http://www.kb.cert.org/vuls/id/402580).

Как обычно, релиз содержит дальнейшие улучшения анонимности и производительности, а также ряд исправлений ошибок. Рекомендуется обновиться.

Пожалуйста, оставляйте отзывы для разработчиков в IRC #i2p или на [forum.i2p2.de](http://forum.i2p2.de/), [присоединяйтесь к проекту](http://www.i2p2.de/getinvolved.html), распространяйте информацию и [сделайте пожертвование](http://www.i2p2.de/donate.html)! Мы по-прежнему ищем помощь с новыми и уже существующими переводами. Пожалуйста, предложите свою помощь в IRC #i2p.

Файлы доступны на [странице загрузки](/downloads/).

Сведения о выпуске:

## Установщик

- Fix installations on 64-bit Windows by upgrading to launch4j 3.0.1 and adding 64-bit dll to izpack.
- The wrapper is still 32-bit only. 64-bit Windows users should start I2P with the 'No Window' icon if the 'Restartable' icon does not work.

## Транспорт

- Better handle clock skews at startup, even if NTP does not work because UDP is blocked or other reasons
- Fix clock skew reporting
- Bootstrap profiles when loading the netDb from disk (broken in 0.7.8)

## Встроенный веб-сервер (Jetty)

- Upgrade to [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) which fixes [this vulnerability](http://www.kb.cert.org/vuls/id/402580).
- Turn on checkAliases for increased protection

## Сетевая база данных

- Add some DOS protection for floodfills
- Increase the number of floodfills again

## Прочее

- Faster startup
- Bug fixes in i2psnark, BOB, clock handling, and more
