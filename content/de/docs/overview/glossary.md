Dieses Glossar definiert gebräuchliche Begriffe, die in der I2P-Dokumentation und in Diskussionen verwendet werden.

## B

**Blockchain** Ein verteiltes Hauptbuch, das Transaktionen zwischen mehreren Parteien effizient sowie auf überprüfbare und dauerhafte Weise aufzeichnen kann.

## D

**Dezentralisierung** Netzwerkarchitektur, die die Abhängigkeit von einer einzelnen Partei vermeidet. Umfasst Peer-to-Peer-, Blockchain-, föderierte und verteilte Technologien.

**Destination** Die kryptografische Identität eines Tunnels. Dies sind die Identitäten von Clients und Servern innerhalb des I2P-Netzwerks.

**DHT (Distributed Hash Table)** Wird in einigen Projekten verwendet, um Peers miteinander zu verbinden, indem Informationen in Form von Schlüssel-Wert-Paaren dezentral gespeichert werden.

**Distributed Systems** Akademisches Thema innerhalb der Informatik, das sich mit dem Design von Computersystemen befasst, die aus vielen einzelnen Computern bestehen, die über ein Netzwerk verbunden sind.

## F

**Federated** Föderation ermöglicht es getrennten Bereitstellungen eines Dienstes, über ein gemeinsames Protokoll miteinander zu kommunizieren.

## H

**Hash** Eine Zahl, die üblicherweise als Zeichenfolge aus Buchstaben und Zahlen dargestellt wird. Sie kann als „Fingerabdruck" dienen, der Daten eindeutig identifiziert.

## I

**I2P** Invisible Internet Project: ein Projekt, das eine Anonymitätsschicht bereitstellen soll, damit Benutzer anonym unter Verwendung einer Reihe von Anwendungen kommunizieren können.

**IP-Adresse** Eine Nummer eines Computers oder Netzwerks, die eindeutig ist und somit zur Adressierung verwendet werden kann.

## L

**LeaseSet** Eine Sammlung von Informationen, die erforderlich sind, um mit einem Client oder Server an einer bestimmten Destination zu kommunizieren.

## P

**Peer-to-Peer (P2P)** Peers stellen einen Teil ihrer Ressourcen direkt anderen Netzwerkteilnehmern zur Verfügung, ohne dass eine zentrale Koordinierung durch Server oder stabile Hosts erforderlich ist.

## R

**Router** Die zentrale I2P-Software, die verschlüsselte Pakete im I2P-Netzwerk weiterleitet.

**RouterIdentity** Eine Sammlung von Informationen, die für die direkte Kommunikation mit einem Router erforderlich sind.

## T

**TCP/UDP** Die beiden grundlegenden Transportprotokolle, die im Internet verwendet werden.

**Tunnel** Ein anonymer Kommunikationspfad zwischen einem Client oder Server und dem I2P-Netzwerk.

## U

**UX** User Experience (Benutzererfahrung), die Gesamterfahrung einer Person bei der Nutzung eines Produkts oder einer Dienstleistung.

## W

**WebRTC** Ein Protokollstandard zum Aufbau von Verbindungen in einem Webbrowser, bei dem Daten direkt zwischen Benutzern übertragen werden.
