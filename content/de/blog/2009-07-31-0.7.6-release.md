I2P-Version 0.7.6 enthält weitere Fehlerbehebungen für Probleme mit der Netzwerkstabilität im Zusammenhang mit dem Aufbau von tunnel und Verbindungslimits. Während die Version 0.7.5 die in den UTC-Abendstunden und an Wochenenden auftretende Überlastung deutlich verringerte, wurde sie nicht vollständig beseitigt; diese Version sollte die Situation weiter verbessern.

Diese Version enthält außerdem neue Konsolen-Themes von dr|z3d, die auf der neuen UI-Konfigurationsseite (Benutzeroberfläche) ausgewählt werden können. Leider sind die neuen Themes nicht für IE-Browser verfügbar.

Die standardmäßige tunnel-Längenvarianz wird von 1 auf 0 geändert. Dies betrifft hauptsächlich exploratory tunnels (Tunnel zur Erkundung des Netzes) und iMule. [Wie auf zzz.i2p erläutert](http://zzz.i2p/topics/260), bringt eine von Null verschiedene Varianz nur sehr geringen zusätzlichen Schutz, verursacht jedoch hohe Ressourcenkosten. Falls Sie Ihre Einstellungen für exploratory tunnels wieder auf die frühere Varianz 1 neu konfigurieren möchten oder auf length 3 variance 0 ändern wollen, können Sie dies auf der Seite config tunnels tun. iMule-Einstellungen ändern Sie am besten direkt in dieser Anwendung.

I2P speichert jetzt temporäre Daten im System-Temp-Verzeichnis (in der Regel /var/tmp unter Linux und %TEMP% unter Windows). Nur bei Neuinstallationen befinden sich die Benutzerdaten in $HOME/.i2p unter Linux/Mac und in %APPDATA%\I2P unter Windows.

Diese Version enthält mehrere Änderungen zur Reduzierung der Ressourcennutzung (Speicher, CPU und Verbindungen) sowie die üblichen Fehlerbehebungen. Ein Upgrade wird empfohlen.

Dies ist die erste Veröffentlichung, die von zzz gebaut und signiert wurde. Nochmals vielen Dank an Complication für seine zahlreichen Beiträge zum Projekt in den vergangenen fünf Jahren. I2P erlebt die rasanteste Entwicklung seit Jahren. Bitte geben Sie den Entwicklern Feedback im IRC #i2p oder auf [forum.i2p2.de](http://forum.i2p2.de) und [machen Sie mit](http://www.i2p2.de/getinvolved.html)!

## Details zur Veröffentlichung:

### Connection Limits and related congestion

- Split the limiting actions into three thresholds (75%, 81%, and 87%) instead of all happening at 80%, for better control, and to save the drastic actions for the higher thresholds.
- Other minor changes, perhaps temporary, to improve tunnel build success.
- Change default tunnel length variance to zero, to help iMule and exploratory build success. As explained elsewhere on this site, this will have a negligible affect on anonymity. Should you wish to keep a variance of 1 for a particular tunnel pool, change it back on configtunnels.jsp or in i2ptunnel, or the iMule config.
- Make shared clients tunnels reduce-on-idle for new installs.

### Verbindungsbeschränkungen und damit verbundene Überlastung

- Expire peer profiles quicker and display less, to reduce memory use.
- Use StringBuilder throughout, faster than StringBuffer
- Reduce stats coalesce frequency, this is a big CPU user
- Allocate much less memory on first startup
- Remove more unused classes

### Ressourcen

- More fixes related to UPnP, recognition of firewalls and changing firewall conditions, and address changes.

### Transport

- Wait until the console is up before launching browser.

### Browser-Startprogramm

- New console, i2psnark, susimail, susidns, and proxy error page themes by dr|zed
- Console themes are selectable on configui.jsp.
- Console themes not available for IE users.
- Fixes for Opera

### Konsole und Designs

- Serve css and image files at the reserved proxy.i2p address, to better display the error pages when the user is on a different machine.

### Proxy-Server

- For new installs, code and data will be split into different directories. Data (router files, config files, i2psnark files, eepsite(I2P Site) files, etc.) will be in ~/.i2p on linux and %APPDATA%\I2P on Windows. The code directory can be read-only to the user (although the user will not be able to update in that case). On linux, the shell scripts i2prouter, runplain.sh, and eepget can be moved to a directory such as /usr/bin. All assumptions that files are in the current working directory are removed. Don't launch the router anymore in the install scripts on linux.
- For existing installs, about the only visible change will be a few temporary files now in the Java temporary directory (e.g. /var/tmp on linux) instead of $I2P.

### Installationsprogramm, getrennte Verzeichnisse, distributionsfreundliche Organisation

- Move the reseeder from the routerconsole to the router, for easier embedding of the router in other code (e.g. android, iMule)
