0.9.7 enthält wichtige Fehlerkorrekturen und Verbesserungen.

Erstmals werden routers der Klasse 'N' (mit mindestens 128 KBytes/sec freigegebener Bandbreite) automatisch zu floodfill (zuvor waren es nur 'O'-routers mit 256 KBps). Dies wird die floodfill-Population erhöhen und zusätzlichen Widerstand gegen bestimmte Angriffe bieten (siehe unten). Floodfill routers verbrauchen nicht viel zusätzliche Bandbreite, neigen jedoch dazu, zusätzlichen Speicher und gleichzeitige Verbindungen zu verwenden. Wenn Sie nicht möchten, dass Ihr router zu floodfill wird, setzen Sie in der erweiterten Konfiguration router.floodfillParticipant=false.

Da wir davon ausgehen, dass das letzte Release die experimentellen Update-via-Torrent-Bugs behoben hat, sollten 3% der routers in diesem Zyklus ihr Update über BitTorrent innerhalb des Netzwerks beziehen.

Die Plugin-Update-Prüfungen, die möglicherweise über mehrere Releases hinweg fehlerhaft waren, wurden behoben. Ihre Plugins sollten, nachdem Sie den router aktualisiert haben, wieder automatisch aktualisiert werden.

Wir haben einen schwerwiegenden Fehler im Streaming-Timer behoben, der mitverantwortlich für häufige IRC-Verbindungsabbrüche war.

Diese Version enthält zusätzliche Gegenmaßnahmen in Reaktion auf das [„praktische Angriffe“-Paper](http://wwwcip.informatik.uni-erlangen.de/~spjsschl/i2p.pdf). Allerdings liegt noch viel Arbeit vor uns, um Sybil-Angriffen auf die floodfills zu widerstehen und Traffic-Analyse an den Gateways und Endpunkten von exploratory tunnels entgegenzuwirken. Es ist eine gute Erinnerung für alle, dass unser Netzwerk noch relativ klein und verwundbar ist. Derzeit empfehlen wir keine Anwendungen, die irgendjemanden ernsthaft in Gefahr bringen würden. Wir werden weiter daran arbeiten, es zu verbessern... bitte helft weiterhin, es bekannt zu machen. Ein größeres Netzwerk ist ein besseres Netzwerk.

## DETAILS ZUR VERÖFFENTLICHUNG

### Anonymity Improvements

- End-to-end encryption of responses to leaseset lookups
- Expand floodfill pool by enabling class 'N' floodfills
- Randomize padding inside encrypted SSU packets
- Preparation for better SSU protocol obfuscation

### Verbesserungen der Anonymität

- Fix newer lease sets not getting stored or published
- Fix classpath bug when used with 4-year-old installations, causing the console not to start
- Fix addressbook database bug preventing update of the reverse index
- Fix i2psnark bug that changed the infohash of torrents created by Robert and fetched via magnet link
- Fix version checking for plugins
- Fix a streaming timer bug causing frequent IRC disconnects (also affects other close-on-idle tunnels)

### Fehlerbehebungen

- Don't install as a service on Windows by default
- Reduce transport idle timeouts
- Reduce tunnels on idle in i2psnark
- Change default in i2ptunnel GUI to 3 hops
- IE 10 support
- Individual expiration times in leases, for efficiency on destinations with a high number of tunnels
- Low-level encryption and XOR speedups
- Jetty 7.6.11
- Tomcat 6.0.37
- Translation updates: Chinese, French, German, Portuguese, Russian, Spanish
- New Turkish translation
- Wrapper 3.5.19 (new installs and PPA only)
- Update GeoIP data (new installs and PPA only)
