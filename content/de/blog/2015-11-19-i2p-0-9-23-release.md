Hallo I2P! Dies ist das erste Release, das von mir (str4d) signiert wurde, nach 49 Releases, die von zzz signiert wurden. Dies ist ein wichtiger Test unserer Redundanz in jeder Hinsicht, auch personell.

## Wartungsarbeiten

Mein Signaturschlüssel ist seit über zwei Jahren (seit 0.9.9) in den router-Updates enthalten, daher sollte dieses Update, wenn Sie eine aktuelle Version von I2P verwenden, genauso einfach sein wie jedes andere Update. Wenn Sie jedoch eine ältere Version als 0.9.9 ausführen, müssen Sie zunächst manuell auf eine aktuelle Version aktualisieren. Sobald Sie manuell aktualisiert haben, wird Ihr router dann wie üblich das 0.9.23-Update finden und herunterladen.

Wenn Sie I2P über einen Paketmanager installiert haben, sind Sie von der Änderung nicht betroffen und können wie gewohnt aktualisieren.

## Aktualisierungsdetails

Die Migration der RouterInfos auf neue, stärkere Ed25519-Signaturen verläuft gut; Schätzungen zufolge hat bereits mindestens die Hälfte des Netzwerks den Schlüssel gewechselt. Diese Version beschleunigt den Schlüsselwechsel. Um die Netzwerkfluktuation zu reduzieren, hat Ihr router bei jedem Neustart eine kleine Wahrscheinlichkeit, auf Ed25519 umzustellen. Wenn er den Schlüssel wechselt, rechnen Sie für ein paar Tage mit einer geringeren Bandbreitennutzung, während er sich mit seiner neuen Identität wieder in das Netzwerk eingliedert.

Beachten Sie, dass dies die letzte Version sein wird, die Java 6 unterstützt. Bitte aktualisieren Sie so bald wie möglich auf Java 7 oder 8. Wir arbeiten bereits daran, I2P mit dem kommenden Java 9 kompatibel zu machen, und ein Teil dieser Arbeit ist in dieser Version enthalten.

Wir haben außerdem einige kleinere Verbesserungen in I2PSnark vorgenommen und eine neue Seite in der routerconsole hinzugefügt, um ältere News-Einträge anzuzeigen.

Wie üblich empfehlen wir, auf diese Version zu aktualisieren. Der beste Weg, die Sicherheit aufrechtzuerhalten und dem Netzwerk zu helfen, besteht darin, die neueste Version auszuführen.
