I2P artık Java 6 veya daha üstünü gerektiriyor. Java 7'ye yükseltmenizi şiddetle tavsiye ederiz. Hâlâ Java 5 kullanıyorsanız, I2P 0.9.12'yi kurmadan önce Java'nızı yükseltmeniz gerekir.

0.9.12, ECDSA ile imzalanmış Destinations (I2P hedef kimlikleri) için başlangıç düzeyinde destek ekler. Delivery Status Messages (teslim onayları) işlenmesine ilişkin çeşitli düzeltmeler içerir ve bu iletiler artık artırılmış güvenlik için uçtan uca şifrelenmiştir.

Jetty 8'e yükselttik. Jetty 8, Jetty 7 ile neredeyse aynıdır; bu nedenle geçmiş Jetty yükseltmelerinde olduğu gibi karmaşık yapılandırma dosyası dönüştürmeleri gerekmiyor. Elle herhangi bir değişiklik yapılmasına gerek olmamalı.

Her zamanki gibi, bu sürüme güncellemenizi öneriyoruz. Güvenliği sürdürmenin ve ağa yardımcı olmanın en iyi yolu en son sürümü çalıştırmaktır.

Mart ayının başlarında, İnternet arama şirketi [DuckDuckGo](https://duckduckgo.com/) yıllık açık kaynak bağış programının bir parçası olarak Invisible Internet Project (I2P)'ye [bağışladı](https://duck.co/blog/foss2014) $5000. Ödül, DuckDuckGo topluluk portalında halktan gelen [aday gösterimleri](https://duck.co/forum/thread/5174/foss-donation-nomations-2014-edition) temel alınarak verildi.

Bu, I2P'nin şimdiye kadar aldığı en büyük tek bağışı oluşturuyor ve internet topluluğunun gizlilik ve güvenliğe artan ilgisini yansıtıyor. Bu finansman, I2P'nin daha fazla kullanıcıya ulaşmasına, geliştirme çalışmalarını genişletmesine ve kodun denetlenmesine yardımcı olacak. Ayrıca I2P geliştiricilerinin New York City'deki [Real-World Cryptography](http://www.realworldcrypto.com/) konferansı gibi etkinliklere katılmasını sağlayacak; burada geliştiriciler, kamuya anonimlik sağlama yönündeki I2P hedefleri doğrultusunda kriptografi uzmanlarıyla bir araya gelip iş birliği yaptılar.

I2P, cömert bağış için Gabriel Weinberg ve DuckDuckGo'ya ve adaylıklardaki desteği için I2P topluluğuna teşekkür eder.

## SÜRÜM AYRINTILARI

### Anonymity Improvements

- Encrypt Delivery Status Messages
- Add preliminary support for ECDSA-signed Destinations
- Add check for replayed NTCP session requests
- Add throttling and blocking checks to streaming ping processing

### Anonimlik İyileştirmeleri

- Fix RouterInfo exchange in NTCP
- Extend timeout for Delivery Status Messages
- Drop streaming messages from recently closed connections
- Fix restarts on Raspberry Pi
- Restore profileOrganizer.sameCountryBonus advanced config
- Fix for jwebcache and i2phex

### Hata Düzeltmeleri

- Jetty 8.1.14.v20131031; Java 6 now required
- Reduce target connection count again to reduce tunnel reject rate further
- Add rate limit for outbound connections at tunnel endpoints
- Add optional inproxy blocking in i2ptunnel
- Use SSU session key for relay request/response when available
- Include HTTP POST data in SYN packet
- Add getopt library for better argument processing
- More removal of Jetty dependencies
- Remove MD5 code, use Java libraries instead
- Change the default addressbook subscription URL
- Translation updates
- Update GeoIP data (new installs and PPA only)
