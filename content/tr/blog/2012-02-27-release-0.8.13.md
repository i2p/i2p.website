0.8.13 sürümü birkaç hata düzeltmesi ve bir-iki yeni özellik içeriyor. Son sürümün performansı önemli ölçüde artırmış olmasından memnunuz ve ağ, süregelen hızlı büyümeye rağmen iyi çalışıyor. Güncelleme yapmanız önerilir.

Dosyalar [indirme sayfasında](/downloads) mevcuttur.

## SÜRÜM AYRINTILARI

Bu, 0.9'dan önceki son sürüm olabilir; 0.9'da Jetty 6'ya geçeceğiz ve basitleştirilmiş bir router konsolu ana sayfası sunacağız. Bir sonraki sürüm hakkında daha fazla bilgi için önümüzdeki haftalarda router konsolunuzdaki haberler bölümünü takip edin.

Bu sürümden itibaren, router, bir router güncellemesinden sonra yeniden başlatıldığında kısa süre içinde eklenti güncellemelerini denetleyip yükleyecektir. Bunu önlemek için, yeniden başlatmadan önce gelişmiş yapılandırmaya `plugins.autoUpdate=false` ekleyin. Ayrıca, istemci yapılandırma sayfasında yeni bir update-all düğmesi bulunmaktadır.

Daha yüksek koruma için, bazı ülkelerdeki router'lar artık otomatik olarak gizli modu etkinleştiriyor. Ancak gizli mod daha düşük performans veya güvenilirlik sunabilir ve hâlâ geliştirme aşamasındadır. Yeniden başlatmadan önce gizli modu devre dışı bırakmak için gelişmiş yapılandırmaya `router.isHidden=false` ekleyin. Daha sonra gizli modu devre dışı bırakmak için ağ yapılandırma sayfasında *IP Configuration* altında *use all auto-detect methods* seçeneğini seçin. Ülkelerin listesi için zzz.i2p üzerindeki başlığa bakın. Listedeki, bir veya iki I2P kullanıcısından fazlasına sahip tek ülke Çin'dir.

## Major Changes

- Check for and download plugin updates upon restarting after a router update
- Routers in certain countries will now automatically enable hidden mode for increased protection
- New Tunnel Wizard for creating tunnels
- A SIGHUP to the wrapper will now initiate a graceful shutdown

## Hata Düzeltmeleri

- Increase read timeout in HTTP Proxy
- Shutdown hooks will now run when router is shutdown externally (i2prouter stop), broken in 0.8.8
- If an external IP address is specified, bind only to that interface
- Fail tunnels we created when we can't contact the adjacent hop
- Prevent races when saving configuration
- For plugins, check min and max Jetty versions; check all version requirements at startup, not just at install
- Fix plugin startup when console is not on port 7657
- Only stop plugins if they are running
- Fix NPE when no network interfaces are present
- Fix eepget exit code on failure
- Improve inbound tunnel building when in hidden mode
- Publish our router info sooner after startup to facilitate inbound tunnel building
- Fix Streamr tunnel registration

## Diğer

- Add icons to buttons in SusMail and SusiDNS
- Move wrapper PID, status, and log files from /tmp to ~/.i2p (Debian packages and new installs only)
- i2prouter graceful (Debian packages and new installs only)
- Increase number of floodfills
- Repack jars in installer to save a little space
- New translation infrastructure for i2prouter script (not enabled yet)
- New Czech translation (thanks Waseihou)
- German, Italian, Polish, Spanish, Swedish, Ukrainian translation updates
