0.9.17 öncelikle bir hata düzeltme sürümüdür, ancak aynı zamanda daha güçlü kriptografik imzalara geçişimizi de sürdürüyor.

Konsolunuzdaki haberler ve en son router sürümü göstergesi için kullanılan haber akışı sistemini, güvenliği artırmak amacıyla RSA 4096-bit anahtarlar kullanan imzalı bir formata taşıdık.

ECDSA mevcutsa yeni eepsites ve sunucular varsayılan olarak ECDSA ile imzalanır. ECDSA mevcut değilse artık konsol yan çubuğunda bir uyarı var. RedHat kullanıcıları için, ECDSA desteği eklemek amacıyla BouncyCastle Provider (bcprov) jar kurulumlarının başarılı olduğuna dair raporlar aldık.

SSU paket bozulması sorunu ve i2p-messenger ile diğer SAM uygulamalarını etkileyen bir SAM hatası da dahil olmak üzere, birkaç ciddi hatayı düzelttik. Bir önceki sürümde eklenen ancak henüz etkinleştirilmemiş olan ön aşama ECDSA router imzaları için birkaç düzeltme var.

Birçoğumuz Aralık ayında Hamburg'daki 31C3'e katılacak. Masamıza uğrayıp merhaba deyin!

Her zamanki gibi, bu sürüme güncellemenizi öneriyoruz. Güvenliği sürdürmenin ve ağa yardımcı olmanın en iyi yolu, en son sürümü çalıştırmaktır.

## SÜRÜM AYRINTILARI

### Changes

- Signed news
- ECDSA default for new server tunnels
- Reseeding now SSL-only by default

### Değişiklikler

- Fix SSU sending corrupt ack-only packets with partial bitfields
- Fix SSU inbound connection fail from non-DSA router
- Don't select incompatible peers if we are a non-DSA router
- Fix EdDSA signature verification bug
- Set I2NP lookup type flags in all cases, not just when a reply tunnel is used
- Stop i2ptunnel server acceptor thread after close
- Fix bug preventing some plugins from stopping completely
- Fix SAM v3 bug causing failures in incoming connections

### Hata Düzeltmeleri

- Add a warning in the console sidebar if ECDSA not supported
- Log warnings for Java 6 that we will eventually require Java 7
- Don't let proxied routers auto-floodfill
- Don't resend SSU acks that are too old
- Don't publish direct info in SSU address if introducers are required
- New default opentrackers in i2psnark
- Add support for specifiying data directory per-torrent in i2psnark
- Changes in streaming accept() error behavior
- Minor blockfile format changes
- New option for persistent random key to preserve peer ordering across restarts
- Translation updates
- Update GeoIP data (new installs and PPA only)
