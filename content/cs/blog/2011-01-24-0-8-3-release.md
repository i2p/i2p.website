Verze 0.8.3 obsahuje několik vylepšení výkonu, včetně snížení počtu vláken a spotřeby paměti a rychlejší komunikace I2CP (klient-router).

Je zde také nová podpora SSL pro Router Console, nová konfigurační stránka reseedu včetně podpory HTTP proxy pro ty, kteří jsou za restriktivními firewally, nová sekce konfigurace I2CP včetně podpory I2CP-over-SSL pro vzdálené klienty, nová sekce konfigurace limitů připojení serveru a blacklistu pro zvýšenou ochranu proti DoS a nová sekce konfigurace HTTP proxy jump serveru, abyste mohli snadno přidat alternativní jump servery. Statistiky jsou nyní ve výchozím nastavení omezené, aby se snížilo využití paměti; plnou sadu statistik lze znovu povolit na stránce konfigurace statistik. Samozřejmě jsou zde také opravy chyb, takže jako obvykle doporučujeme aktualizaci.

Prosím, pomozte rozvoji sítě. [Zapojte se](http://www.i2p2.de/getinvolved.html), šiřte povědomí a [přispějte](http://www.i2p2.de/donate.html)! Pokud najdete chybu, zadejte prosím hlášení na [gitlab](http://i2pgit.org/i2p-hackers/i2p.i2p/issues). Stále hledáme pomoc s novými i stávajícími překlady. Prosím, přihlaste se jako dobrovolníci na IRC #i2p.

Soubory jsou k dispozici na [stránce ke stažení](/downloads/).

**Podrobnosti o vydání:**

## Nové funkce

* SSL Console
* SSL I2CP
* Jump list, blacklist, and connection limits now on i2ptunnel edit forms
* Smarter update handler that tries multiple sources
* New reseed configuration page for those behind restrictive firewalls
* New I2CP configuration form

## Snížení spotřeby zdrojů

* More efficient I2CP for internal clients
* Less threads, better thread pooling
* Adjust thread count based on available memory
* Change full stats default to false
* Caching of several data structures
* Naming lookup support over existing I2CP sessions

## Opravy chyb

* Fix i2psnark buttons in Firefox 4.0b and IE
* Restore library functions required by Syndie
