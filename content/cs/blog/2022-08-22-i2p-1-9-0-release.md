## Podrobnosti aktualizace

Poslední tři měsíce jsme s malým počtem dobrovolných testerů pracovali na našem novém transportním protokolu UDP "SSU2". Toto vydání dokončuje implementaci, včetně přeposílání a testování peerů. Ve výchozím nastavení jej zapínáme pro platformy Android a ARM a náhodně u malého procenta dalších routers. To nám umožní v následujících třech měsících provést mnohem více testování, dokončit funkci migrace spojení a opravit zbývající problémy. Plánujeme jej zapnout pro všechny v příštím vydání naplánovaném na listopad. Není nutné žádné ruční nastavení. Samozřejmě je v tomto vydání také obvyklá sada oprav chyb. Také jsme přidali automatický detektor deadlocku (vzájemného zablokování), který už odhalil vzácný deadlock, jenž je nyní opraven.

Jako obvykle doporučujeme, abyste aktualizovali na tuto verzi. Nejlepším způsobem, jak udržovat bezpečnost a pomoci síti, je provozovat nejnovější verzi.

## PODROBNOSTI VYDÁNÍ

**Změny**

- Add deadlock detector
- Periodically send our RI to connected peers
- SSU MTU/PMTU improvements and fixes
- SSU2 base protocol fixes and improvements
- SSU2 peer test and relay implementation
- SSU2 published address fixes
- SSU2: Enable for Android, ARM, and a small portion of others at random

**Opravy chyb**

- Clock: Fix deadlock after clock shift
- Debian: Apparmor profile fixes
- Don't allow family key errors to crash router
- Fix EC family key loading on Android
- Fix EdDSA key loading on Java 15+
- i2psnark: Fix DHT not restarting after router restart
- OSX: Prevent hangs at shutdown after dock right-click quit
- SSU: Fix publishing of MTU in addresses without IPs
- SSU: Fix rare HMAC NPE
- SusiDNS CSS fixes
- Transport: Improve processing after message delivery failure
- UPnP: Don't briefly bind to all addresses at startup

**Ostatní**

- Set outproxy to exit.stormycloud.i2p (new installs only)
- Disable SSU introductions on Android
- API version: 0.9.55
- New translation: Spanish (Argentina)
- Translation updates
