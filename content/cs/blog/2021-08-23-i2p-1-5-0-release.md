## Podrobnosti aktualizace

Ano, je to tak, po 9 letech vydávání řady 0.9.x přecházíme přímo z 0.9.50 na 1.5.0. To neznamená zásadní změnu API ani tvrzení, že je vývoj nyní dokončen. Je to pouze uznání téměř 20 let práce na zajištění anonymity a bezpečnosti našim uživatelům.

Toto vydání dokončuje implementaci menších zpráv pro sestavení tunnelů, aby se snížilo využití šířky pásma. Pokračujeme v přechodu routerů sítě na šifrování X25519. Samozřejmě nechybí ani četné opravy chyb a vylepšení výkonu.

Jako obvykle doporučujeme, abyste aktualizovali na tuto verzi. Nejlepší způsob, jak zajistit bezpečnost a pomoci síti, je používat nejnovější verzi.

## PODROBNOSTI VYDÁNÍ

**Změny**

- RRD4J 3.8
- Tunnels: Finish support for new build messages (proposal 157)
- Updates: Support for .dmg and .exe updates

**Opravy chyb**

- Console: Fix generation of SSL keys on Java 17
- i2psnark: Fix autostart for magnets
- Router: Fix rare deadlock in publishing our RI
- SSU: Fix handling of bad peer test responses
- UPnP: IPv6 fixes

**Ostatní**

- Jetty: Improve sort in directory listings
- Jetty: Add X-I2P-Location header
- Router: Increase probability to rekey to ECIES
- Streaming: Performance improvements for low-latency connections
- Translation updates
