I2P verze 0.7.7 obsahuje několik vylepšení anonymity. Použití relačních klíčů pro šifrování bylo kompletně přepracováno a I2PSnark nyní pro komunikaci s trackerem používá své vlastní tunnels. Práce na vylepšování konzole routeru pokračují, včetně souladu se standardy HTML a lepší podpory UTF-8.

Práce také pokračují na zvyšování rychlosti, a to jak snižováním počtu přenášených klíčů relace a zlepšováním metod zahazování zpráv při přetížení, tak snižováním počtu zahozených zpráv u routerů s vysokou šířkou pásma. Pro ty z vás, kteří by chtěli automatické aktualizace na nejnovější vývojové verze na samotné špičce vývoje, je k dispozici nová možnost nepodepsané aktualizace.

Toto vydání také obsahuje několik změn pro snížení využití paměti a CPU a velkou řadu oprav chyb. Doporučujeme aktualizovat. Prosíme, dejte vývojářům zpětnou vazbu na IRC #i2p nebo na [forum.i2p2.de](http://forum.i2p2.de) a [zapojte se](http://www.i2p2.de/getinvolved.html)!

## Podrobnosti o vydání:

### Resources

- Redesign bloom filters, saves 6-8MB heap.
- Reduce PRNG buffers for I2PAppContext, saves 3.5MB heap.
- Remove more unused classes and code, more Java 5 cleanups
- Reduce connection limits slightly.

### Zdroje

- Improve the random early discarder, to be smarter - will hopefully improve tunnel speeds
- Increase the size of the Bloom filter for high bandwidth routers (>= 512KBps share) to reduce packet discards

### Tunnels

- Use multiple Session Key Managers for anonymity benefits
- Reduce the number of Session Tags sent out at once, for efficiency - should reduce stalls on long-lived connections

### Kryptografie

- Lots of HTML cleanups and fixes
- UTF-8 fixes
- New error page

### Konzole

- New unsigned update option
- Verify zip file integrity at startup

### Aktualizace

- Use snark's tunnels rather than the HTTP Proxy for announces and torrent fetches, for efficiency and anonymity.
