Début 2009, peu après notre participation au 25C3, nous avons publié la version 0.7 du réseau anonyme I2P. Au cours des 18 mois suivants, au fil de 15 versions, nous avons travaillé à améliorer la convivialité du logiciel I2P et à en accroître les performances, la sécurité et la fiabilité - tout en faisant face à une croissance sans précédent du réseau.

Pour saluer ces réalisations, nous annonçons le lancement de la série 0.8. La version 0.8 est pleinement compatible avec toutes les versions 0.7.x. Cette version contient plusieurs corrections de bogues, une nouvelle traduction en néerlandais et la finalisation de la traduction allemande. Comme toujours, les utilisateurs sont encouragés à mettre à jour.

Quelques faits marquants visibles de la série de versions 0.7.x au cours de l’année et demie écoulée :

- Automatically generated Base32 destination names, reducing reliance on host names.
- Themable router console with GeoIP display.
- Plugin system for easy installation and maintenance of user-generated applications. For example I2P-bote, distributed serverless anonymous e-mail.
- Translation infrastructure for the router console, and translation into several languages.
- Many improvements leading to less CPU and memory usage.
- Network improvements for scalability, now you can invite all your friends into cipherspace!

Plusieurs améliorations importantes sous le capot :

- Redesign of the floodfill infrastructure for network database storage, by implementing a simplified Kademlia storage system, and an increase in floodfill routers from 5 to almost 100, for increased reliability and scalability.
- Encryption of queries and stores to floodfill routers for increased security.
- Separate session key managers for each local destination for increased security.
- Multi-layered system of limiting connections to peers.
- New, smaller tunnel build message to reduce the chance it will be dropped, and increase build success rates.
- Several bug fixes and improvements in the streaming library to speed up connections dramatically.
- Transition to Java 5 concurrent data structures for a significant reduction in lock contention.
- Transition to Java 5 coding styles, a significant increase in javadoc documentation, and elimination of unused code for maintanability.
- Split directory structure for ease of use on multi-user systems and easier packaging.
- Clock skew system redesign, with implementation of NTP-style clock adjustment.
- Adjustment of several parameters based on the speed and maximum memory of the router, to optimize speed and memory usage.
- UPnP support to open firewall ports.
- An uncountable number of bugfixes and improvements that enhance security, anonymity, reliability, performance, scalability, and the user experience.

Dans [l’entretien de zzz avec gulli.com l’année dernière](http://www.gulli.com/news/i2p-an-anonymous-network-2009-03-09/), il a déclaré que « la vie privée est gravement et de plus en plus menacée partout dans le monde ». Il est clair que cette tendance s’accélère aussi bien dans les nations « opprimées » que dans celles qui sont nominalement « libres ». Alors que nous nous préparons à assister à [la conférence HOPE à New York](http://thenexthope.org/), avec de nombreuses conférences sur la vie privée, nous élaborerons des plans pour continuer à améliorer I2P et le faire connaître.

Vous pouvez aider, comme toujours, en rejoignant le réseau et en partageant votre bande passante. Donnez votre avis aux développeurs sur IRC #i2p ou sur [forum.i2p2.de](http://forum.i2p2.de/), [impliquez-vous](http://www.i2p2.de/getinvolved.html), faites passer le mot et [faites un don](http://www.i2p2.de/donate.html) ! Nous sommes toujours à la recherche d'aide pour les traductions nouvelles et existantes. Merci de vous porter volontaire sur IRC #i2p.
