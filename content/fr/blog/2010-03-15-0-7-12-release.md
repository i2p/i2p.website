La version 0.7.12 inclut la prise en charge des plugins créés par les utilisateurs. Ces plugins peuvent inclure des applications web de la console, des thèmes, des traductions ou des programmes autonomes. Certains plugins sont déjà disponibles pour des tests. Nous espérons que cette prise en charge permettra le développement rapide d’applications i2p innovantes.

Cette version corrige le bogue de page blanche lorsqu’un eepsite(I2P Site) est injoignable et améliore également la gestion des décalages d’horloge et des changements d’adresse IP. Elle ajoute la prise en charge d’un nouveau message de construction de tunnel, plus petit, qui sera testé dans cette version et activé dans la prochaine version.

Comme d'habitude, cette version apporte davantage d'améliorations en matière d'anonymat et de performances, des mises à jour de traduction, ainsi qu'un ensemble d'autres corrections de bogues. La mise à niveau est recommandée.

Les six derniers mois de développement ont considérablement augmenté les performances et la capacité du réseau. Aidez-nous à développer le réseau. Merci de faire part de vos retours aux développeurs sur IRC #i2p ou sur [forum.i2p2.de](http://forum.i2p2.de/), de [vous impliquer](http://www.i2p2.de/getinvolved.html), de faire passer le mot et de [faire un don](http://www.i2p2.de/donate.html) ! Nous sommes toujours à la recherche d'aide pour les traductions, nouvelles comme existantes. Merci de vous porter volontaire sur IRC #i2p.

Les fichiers sont disponibles sur la [page de téléchargement](/downloads/).

Détails de la version :

## Greffons

- New support for cryptographically-signed user-generated i2p plugins. These plugins may provide new console web applications, themes, translations, web sites, or other applications.

## I2PSnark

- Ignore non-i2p trackers in a torrent, to "convert" a non-i2p torrent to an i2p torrent using i2p open trackers
- Add javascript confirmation to remove and delete buttons

## Transport

- Implement NTP-style clock slewing to converge the router clock without jumps
- Implement clock updates based on peer average, for clock convergence even if NTP updates fail
- Fix IP change recognition for routers that are not firewalled
- Implement address costs, so a router can advertise its preferred transport
- Fix the router gradually losing geoip references
- Improve performance by removing global locks
- Remove some unused threads

## Tunnel Building

- Implement a new, smaller tunnel build message, to reduce bandwidth and improve build success. The new message will be tested in this release and enabled in the next release.
- Accept tunnel build replies even if they are received late, to improve build success
- Improve performance by removing global locks

## Proxy HTTP

- Fix bug causing blank page instead of error page on connect timeout
- Change Host: header from B64 to B32 to reduce the request size
- Eliminate redundant lookups

## Console

- Fix repeated console password prompts
- Russian, Chinese, and French translation updates

## Base de données du réseau

- Increase the number of floodfills again

## Autres

- Support unpack200 in the updater for smaller update packages and plugins
- More changes for performance
- Take library jars out of susidns and router console
- A new SOCKS IRC tunnel type, to provide IRC filtering and server selection in a single client
