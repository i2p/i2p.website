## Détails de la mise à jour

La version 0.9.44 contient un correctif important pour un problème de déni de service dans la gestion des nouveaux types de chiffrement par les services cachés. Tous les utilisateurs devraient mettre à jour dès que possible.

La version inclut une prise en charge initiale du nouveau chiffrement de bout en bout (proposition 144). Les travaux se poursuivent sur ce projet, qui n'est pas encore prêt à l'emploi. Il y a des changements sur la page d'accueil de la console, et de nouveaux lecteurs multimédias HTML5 intégrés dans i2psnark. Des correctifs supplémentaires pour les réseaux IPv6 derrière un pare-feu sont inclus. Des correctifs pour la construction de tunnel devraient entraîner un démarrage plus rapide pour certains utilisateurs.

Comme d’habitude, nous vous recommandons de mettre à jour vers cette version. La meilleure façon de maintenir la sécurité et d’aider le réseau est d’exécuter la dernière version.

## DÉTAILS DE LA VERSION


**Modifications**

- Console home page changes
- Initial code for new encryption (proposal 144)
- Router changes to support multiple encryption types
- HTML5 players in i2psnark
- Audio playlist in i2psnark
- Change Windows data location to %LOCALAPPDIR% (new installs only)

**Corrections de bogues**

- Fix AppArmor config for Java 11
- Fix handling of unsupported encryption types
- IPv6 firewalled fixes
- Fix long stop delay for some plugins
- Fix custom icons for some plugins
- Fix exploratory tunnel build issues causing long startup times
- Prevent multiple zero-hop tunnels in a pool
- Recognize gettext 0.20 in build scripts

**Autre**

- Don't check clove ID in Bloom filter
- Reduce SessionTag memory usage
- Change default sig type to EdDSA for Android
- Translation updates
- Update GeoIP data
