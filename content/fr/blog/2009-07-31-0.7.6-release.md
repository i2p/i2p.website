La version 0.7.6 d’I2P contient des correctifs supplémentaires pour des problèmes de stabilité du réseau liés à la construction des tunnels et aux limites de connexion. Bien que la version 0.7.5 ait fortement réduit la congestion qui survenait en soirée et le week-end (UTC), elle ne l’a pas éliminée ; cette version devrait améliorer encore la situation.

Cette version inclut également de nouveaux thèmes de console conçus par dr|z3d, qui peuvent être sélectionnés sur la nouvelle page de configuration de l’interface utilisateur. Désolé, les nouveaux thèmes ne sont pas disponibles pour les navigateurs Internet Explorer.

La variance par défaut de la longueur des tunnels est passée de 1 à 0. Cela affecte principalement les tunnels exploratoires et iMule. [Comme expliqué sur zzz.i2p](http://zzz.i2p/topics/260), une variance non nulle ajoute très peu de protection, pour un coût élevé en ressources. Si vous souhaitez reconfigurer vos paramètres des tunnels exploratoires à l’ancienne variance de 1, ou passer à longueur 3 variance 0, vous pouvez le faire sur la page de configuration des tunnels. Les paramètres d’iMule sont de préférence modifiés directement dans cette application.

I2P stocke désormais les données temporaires dans le répertoire temporaire du système (généralement /var/tmp sous Linux et %TEMP% sous Windows). Pour les nouvelles installations uniquement, les données utilisateur se trouvent dans $HOME/.i2p sous Linux/Mac et %APPDATA%\I2P sous Windows.

Cette version contient plusieurs modifications visant à réduire l’utilisation des ressources (mémoire, CPU et connexions) ainsi que le lot habituel de corrections de bogues. La mise à jour est recommandée.

C’est la première version compilée et signée par zzz. Merci encore à Complication pour ses nombreuses contributions au projet au cours des cinq dernières années. I2P connaît son développement le plus rapide depuis des années. Donnez aux développeurs vos retours sur IRC #i2p ou sur [forum.i2p2.de](http://forum.i2p2.de) et [impliquez-vous](http://www.i2p2.de/getinvolved.html) !

## Détails de la version:

### Connection Limits and related congestion

- Split the limiting actions into three thresholds (75%, 81%, and 87%) instead of all happening at 80%, for better control, and to save the drastic actions for the higher thresholds.
- Other minor changes, perhaps temporary, to improve tunnel build success.
- Change default tunnel length variance to zero, to help iMule and exploratory build success. As explained elsewhere on this site, this will have a negligible affect on anonymity. Should you wish to keep a variance of 1 for a particular tunnel pool, change it back on configtunnels.jsp or in i2ptunnel, or the iMule config.
- Make shared clients tunnels reduce-on-idle for new installs.

### Limites de connexion et congestion associée

- Expire peer profiles quicker and display less, to reduce memory use.
- Use StringBuilder throughout, faster than StringBuffer
- Reduce stats coalesce frequency, this is a big CPU user
- Allocate much less memory on first startup
- Remove more unused classes

### Ressources

- More fixes related to UPnP, recognition of firewalls and changing firewall conditions, and address changes.

### Transport

- Wait until the console is up before launching browser.

### Lanceur de navigateur

- New console, i2psnark, susimail, susidns, and proxy error page themes by dr|zed
- Console themes are selectable on configui.jsp.
- Console themes not available for IE users.
- Fixes for Opera

### Console et thèmes

- Serve css and image files at the reserved proxy.i2p address, to better display the error pages when the user is on a different machine.

### Proxy

- For new installs, code and data will be split into different directories. Data (router files, config files, i2psnark files, eepsite(I2P Site) files, etc.) will be in ~/.i2p on linux and %APPDATA%\I2P on Windows. The code directory can be read-only to the user (although the user will not be able to update in that case). On linux, the shell scripts i2prouter, runplain.sh, and eepget can be moved to a directory such as /usr/bin. All assumptions that files are in the current working directory are removed. Don't launch the router anymore in the install scripts on linux.
- For existing installs, about the only visible change will be a few temporary files now in the Java temporary directory (e.g. /var/tmp on linux) instead of $I2P.

### Programme d'installation, Répertoires séparés, Organisation compatible avec les distributions

- Move the reseeder from the routerconsole to the router, for easier embedding of the router in other code (e.g. android, iMule)
