I2P version 0.6.5
=====================

La version 0.6.5 introduit de nouveaux composants, supprime certains anciens (comme l’ancien transport TCP) et a été optimisée pour offrir de meilleures performances.

Le protocole BOB (Basic Open Bridge) est introduit, à l’usage des applications clientes qui ne peuvent pas importer directement les bibliothèques I2CP. Cela rend obsolète l’ancien protocole SAM auparavant utilisé dans de tels cas. Pour l’instant toutefois, BOB n’est pas encore démarré automatiquement sur les nouvelles installations, et SAM reste actif sur les anciennes installations.

Un code amélioré devrait mieux prévenir la congestion en rejetant de manière probabiliste du trafic en transit, et se comporter également mieux lorsque la congestion survient. Le floodfill NetDB devrait fonctionner de manière plus fiable, la bibliothèque de streaming devrait choisir des tailles de message plus adaptées, offrir une fonction de délai d’expiration de socket, et les travaux se poursuivent sur le mode de fonctionnement "hidden" pour les I2P routers.

- From this release onward, I2P requires Java 1.5 or higher.

Si vous ne connaissez pas votre version de Java, vous pouvez vérifier en ouvrant une fenêtre de terminal ou une invite de commandes, puis en entrant la commande "java -version". Si vous avez une version de Java plus ancienne installée, veuillez la mettre à jour d'abord !

Cette version est rétrocompatible : si vous avez déjà la version 0.6.1.31 ou supérieure installée, elle reconnaîtra la signature du fichier de mise à jour signé et vous pourrez mettre à jour automatiquement.
