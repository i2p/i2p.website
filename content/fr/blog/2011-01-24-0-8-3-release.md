La version 0.8.3 contient plusieurs améliorations des performances, notamment une réduction du nombre de threads et de l’utilisation de la mémoire, ainsi qu’une communication I2CP (client-router) plus rapide.

Il y a également une nouvelle prise en charge SSL pour la console du router, une nouvelle page de configuration du reseed (réamorçage) incluant la prise en charge d’un proxy HTTP pour ceux qui se trouvent derrière des pare-feux restrictifs, une nouvelle section de configuration I2CP incluant la prise en charge d’I2CP-over-SSL pour les clients distants, une nouvelle section de configuration des limites de connexions serveur et de la liste noire pour une protection DoS renforcée, ainsi qu’une nouvelle section de configuration des jump servers (serveurs de saut) du proxy HTTP afin que vous puissiez facilement ajouter des jump servers alternatifs. Les statistiques sont désormais limitées par défaut afin de réduire l’utilisation de la mémoire ; l’ensemble complet des statistiques peut être réactivé sur la page de configuration des statistiques. Il y a aussi des corrections de bogues, bien sûr ; comme d’habitude, la mise à niveau est recommandée.

Aidez-nous à développer le réseau. [Participez](http://www.i2p2.de/getinvolved.html), faites passer le mot, et [faites un don](http://www.i2p2.de/donate.html) ! Si vous trouvez un bug, veuillez soumettre un rapport sur [gitlab](http://i2pgit.org/i2p-hackers/i2p.i2p/issues). Nous recherchons toujours de l'aide pour les nouvelles traductions et celles déjà existantes. Merci de vous proposer sur IRC #i2p.

Les fichiers sont disponibles sur la [page de téléchargement](/downloads/).

**Détails de la version:**

## Nouvelles fonctionnalités

* SSL Console
* SSL I2CP
* Jump list, blacklist, and connection limits now on i2ptunnel edit forms
* Smarter update handler that tries multiple sources
* New reseed configuration page for those behind restrictive firewalls
* New I2CP configuration form

## Réduction des ressources

* More efficient I2CP for internal clients
* Less threads, better thread pooling
* Adjust thread count based on available memory
* Change full stats default to false
* Caching of several data structures
* Naming lookup support over existing I2CP sessions

## Corrections de bogues

* Fix i2psnark buttons in Firefox 4.0b and IE
* Restore library functions required by Syndie
