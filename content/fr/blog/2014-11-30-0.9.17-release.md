La version 0.9.17 est principalement une version corrective, mais elle poursuit également notre migration vers des signatures cryptographiques plus robustes.

Nous avons migré le système de flux d’actualités utilisé pour les actualités sur votre console et l’indication de la dernière version du router vers un format signé utilisant des clés RSA 4096 bits pour une sécurité renforcée.

Les nouveaux eepsites et serveurs seront signés avec ECDSA par défaut, si ECDSA est disponible. Un avertissement apparaît désormais dans la barre latérale de la console si ECDSA n’est pas disponible. Pour les utilisateurs de RedHat, nous avons des retours d’installations réussies du JAR BouncyCastle Provider (bcprov) afin d’ajouter la prise en charge d’ECDSA.

Nous avons corrigé plusieurs bogues graves, dont un problème de corruption de paquets SSU, ainsi qu’un bogue SAM affectant i2p-messenger et d’autres applications SAM. Plusieurs correctifs portent sur les signatures ECDSA préliminaires du router, ajoutées dans la dernière version mais pas encore activées.

Beaucoup d’entre nous assisteront au 31C3 à Hambourg en décembre. Passez à notre table nous dire bonjour !

Comme d'habitude, nous vous recommandons de mettre à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'utiliser la dernière version.

## DÉTAILS DE LA VERSION

### Changes

- Signed news
- ECDSA default for new server tunnels
- Reseeding now SSL-only by default

### Modifications

- Fix SSU sending corrupt ack-only packets with partial bitfields
- Fix SSU inbound connection fail from non-DSA router
- Don't select incompatible peers if we are a non-DSA router
- Fix EdDSA signature verification bug
- Set I2NP lookup type flags in all cases, not just when a reply tunnel is used
- Stop i2ptunnel server acceptor thread after close
- Fix bug preventing some plugins from stopping completely
- Fix SAM v3 bug causing failures in incoming connections

### Corrections de bogues

- Add a warning in the console sidebar if ECDSA not supported
- Log warnings for Java 6 that we will eventually require Java 7
- Don't let proxied routers auto-floodfill
- Don't resend SSU acks that are too old
- Don't publish direct info in SSU address if introducers are required
- New default opentrackers in i2psnark
- Add support for specifiying data directory per-torrent in i2psnark
- Changes in streaming accept() error behavior
- Minor blockfile format changes
- New option for persistent random key to preserve peer ordering across restarts
- Translation updates
- Update GeoIP data (new installs and PPA only)
