Après la publication d’I2P 2.2.0, avancée pour accélérer les mesures d’atténuation face aux attaques DDOS, nous avons découvert quelques problèmes émergents qui ont rendu nécessaire la production et la publication de nouveaux paquets. Cette version corrige un problème sous Ubuntu Lunar et Debian Sid où la console du router était inaccessible lors de l’utilisation d’une version mise à jour du paquet jakarta. Les paquets Docker ne lisaient pas correctement les arguments, rendant les fichiers de configuration inaccessibles. Ce problème a également été résolu. Le conteneur Docker est désormais également compatible avec Podman.

Cette version synchronise les traductions avec transifex et met à jour la base de données GeoIP.

Comme d'habitude, nous vous recommandons de mettre à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'utiliser la dernière version.

## Détails

### Changes

- Fix missing Java options in docker/rootfs/startapp.sh
- Detect when running in Podman instead of regular Docker
- Update Tor Browser User-Agent String
- Update local GeoIP database
- Remove invalid signing keys from old installs
- Update Tomcat version in Ubuntu Lunar and Debian Sid
