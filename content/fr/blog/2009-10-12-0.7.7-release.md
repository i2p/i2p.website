La version 0.7.7 d’I2P comprend plusieurs améliorations en matière d’anonymat. L’utilisation de clés de session pour la cryptographie a été entièrement repensée, et I2PSnark utilise désormais ses propres tunnels pour la communication avec le tracker. Le travail se poursuit pour améliorer la console du router, y compris la conformité aux normes HTML et une meilleure prise en charge de l’UTF-8.

Les travaux se poursuivent également pour améliorer les débits, à la fois en réduisant le nombre de clés de session transférées, en améliorant les méthodes de rejet des messages en cas de surcharge, et en réduisant les rejets par les routers à haute bande passante. Une nouvelle option de mise à jour non signée est disponible pour celles et ceux qui souhaitent des mises à jour automatiques vers les toutes dernières versions de développement.

Cette version contient également plusieurs changements visant à réduire l’utilisation de la mémoire et du CPU, ainsi qu’un grand nombre de corrections de bogues. La mise à jour est recommandée. Veuillez faire part de vos retours aux développeurs sur IRC #i2p ou sur [forum.i2p2.de](http://forum.i2p2.de) et [participez](http://www.i2p2.de/getinvolved.html) !

## Détails de la version:

### Resources

- Redesign bloom filters, saves 6-8MB heap.
- Reduce PRNG buffers for I2PAppContext, saves 3.5MB heap.
- Remove more unused classes and code, more Java 5 cleanups
- Reduce connection limits slightly.

### Ressources

- Improve the random early discarder, to be smarter - will hopefully improve tunnel speeds
- Increase the size of the Bloom filter for high bandwidth routers (>= 512KBps share) to reduce packet discards

### Tunnels

- Use multiple Session Key Managers for anonymity benefits
- Reduce the number of Session Tags sent out at once, for efficiency - should reduce stalls on long-lived connections

### Cryptographie

- Lots of HTML cleanups and fixes
- UTF-8 fixes
- New error page

### Console

- New unsigned update option
- Verify zip file integrity at startup

### Mises à jour

- Use snark's tunnels rather than the HTTP Proxy for announces and torrent fetches, for efficiency and anonymity.
