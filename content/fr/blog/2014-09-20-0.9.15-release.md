La version 0.9.15 ajoute la prise en charge préliminaire des signatures Ed25519 EdDSA. Elle inclut un nouveau back-end de configuration persistant pour i2psnark et corrige plusieurs problèmes dans la gestion des noms de fichiers par i2psnark. Plusieurs améliorations visent à accélérer SAM. Les plugins prennent désormais en charge des signatures plus fortes dans le format de fichier su3. L’installation des plugins via la console, qui avait été désactivée dans la version 0.9.14, est réactivée.

Nous prenons en charge les signatures ECDSA depuis la 0.9.12, et nous souhaiterions commencer à utiliser ECDSA par défaut. Malheureusement, certains d’entre vous utilisent encore des versions I2P plus anciennes, et pour d’autres, leur distribution ou l’environnement d’exécution Java ne prend pas en charge ECDSA. Il est signalé que les distributions Red Hat (RHEL, Fedora) sont dépourvues d’ECDSA. Certains ont corrigé les problèmes Java en passant de Java 6 à Java 7 ; d’autres ont réussi en installant les "unlimited strength policy files" (fichiers de stratégie à force illimitée). Nous avons ajouté des informations sur les fonctions cryptographiques manquantes dans le fichier journal et sur la page /logs de la console. Après la mise à jour vers la version 0.9.15, veuillez vérifier si la prise en charge d’ECDSA est absente et tentez de la corriger si nécessaire. Ceci est particulièrement important pour celles et ceux qui exploitent des eepsites et des services populaires.

Comme d'habitude, nous vous recommandons de mettre à jour vers cette version. La meilleure façon de maintenir la sécurité et d'aider le réseau est d'exécuter la dernière version.

## DÉTAILS DE LA VERSION

### Changes

- Add support for Ed25519 signatures
- i2psnark move to separate config file for each torrent to better support per-torrent settings
- Add i2psnark support for data outside the i2psnark/ directory
- Enable stronger signatures (su3 format) for plugins
- Speed up SSU introductions by responding to hole punch messages
- Several improvements in SAM efficiency

### Modifications

- Form submission fixes in the console and i2psnark
- Streaming fixes for long signatures
- i2psnark fixes for file name character mapping when seeding
- I2PTunnel fixes stopping client tunnels
- I2PTunnel fix updating options on a running delay-open client tunnel

### Corrections de bogues

- Re-enable plugin installation via the console, removed in 0.9.14
- i2psnark now remembers uploaded count across restarts
- i2psnark increase max piece size to 8 MB
- i2psnark several UI fixes
- Prohibit SSU peer test requests unless a connection is established
- i2ptunnel add support for local SSL connections for standard and IRC client tunnels
- Console and log warnings for unavailable crypto
- More consistent routing for Delivery Status Messages to reduce network connections
- Disable external entities in UPnP XML parser
- More escaping and cleanups in forms and messages
- Translation updates
- Update GeoIP data (in both new installs and updates)
