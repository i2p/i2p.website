La 0.9.19 incluye varias correcciones y mejoras para el rendimiento de floodfill. Muchos de ustedes observaron un alto uso de CPU después del lanzamiento de la 0.9.18. Esto fue causado por una combinación de un mayor uso de cifrado, la gran afluencia de usuarios de Vuze a la red, una reducción de los floodfills debido a requisitos de rendimiento más estrictos y algunos errores antiguos. Las cosas deberían estar mucho mejor una vez que la mayor parte de la red se haya actualizado. Como siempre, la mejor manera de reducir el uso de CPU es bajar los límites de ancho de banda.

También hemos añadido nuevas formas de hacer reseed (proceso de obtención de nodos iniciales) manualmente, y de generar un archivo de reseed que puedes compartir fácilmente con quienes lo necesiten. Consulta la página de configuración de reseed en la consola del router para obtener más información.

Como de costumbre, recomendamos que actualice a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es usar la versión más reciente.

## DETALLES DE LA VERSIÓN

### Changes

- Floodfill performance improvements
- Easier ways to reseed manually from a file or URL
- New way to export reseed data for others
- Support for installing plugin from file

### Cambios

- Fixes for high CPU usage in floodfills
- i2ptunnel locking fixes
- Fixes for read timeout handling in streaming
- Fix changing i2psnark data directory on Windows
- Fix multiple SSL outproxies in HTTP client

### Correcciones de errores

- Update to UPnP library version 3.0
- Improve tracking of floodfill lookup success
- Direct router info lookups if connected to floodfill
- Auto-adjustment of i2psnark tunnel quantity
- Increase exploratory tunnel quantity when floodfill
- Increase min and default bandwidth for i2psnark
- Improved strategies for dropping jobs on high job lag to prevent overload
- Drop tunnel build requests on high job lag
- Increase allowed clock skew in I2CP
- New HTTP error page when the server resets the connection
- Require ECDSA support for floodfill
- Republish router info faster when capabilities change
- Better feedback in console for reseed errors
- Apache Tomcat 6.0.43
- Translation updates
- Update GeoIP data (new installs and PPA only)
