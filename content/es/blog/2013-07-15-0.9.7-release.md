La versión 0.9.7 incluye importantes correcciones de errores y mejoras.

Por primera vez, los routers de clase 'N' (aquellos con un mínimo de 128 KBytes/sec de ancho de banda compartido) se convertirán automáticamente en floodfill (anteriormente solo eran routers 'O' con 256 KBps). Esto aumentará la población de floodfill para ofrecer mayor resistencia a ciertos ataques (ver más abajo). Los routers floodfill no consumen mucho ancho de banda adicional, pero sí tienden a usar memoria adicional y conexiones concurrentes. Si no desea que su router se convierta en floodfill, establezca la configuración avanzada router.floodfillParticipant=false.

Como creemos que la última versión corrigió los errores experimentales de actualización vía torrent, el 3% de los routers debería actualizarse a través de bittorrent dentro de la red en este ciclo.

Se han corregido las comprobaciones de actualización de los complementos, que posiblemente han estado fallando durante varias versiones. Sus complementos deberían volver a actualizarse automáticamente tras actualizar el router.

Corregimos un importante error en el temporizador de streaming que contribuía a desconexiones frecuentes en IRC.

This release contains additional mitigations for the ["practical attacks" paper](http://wwwcip.informatik.uni-erlangen.de/~spjsschl/i2p.pdf). However, we have a lot more work to do to resist Sybil attacks on the floodfills, and resist traffic analysis at the gateways and endpoints of exploratory tunnels. It's a good reminder for everybody that our network is still relatively small and vulnerable. We don't currently recommend any uses that would put anybody in serious jeopardy. We'll keep working to improve it... please keep working to spread the word. A bigger network is a better network.

## DETALLES DE LA VERSIÓN

### Anonymity Improvements

- End-to-end encryption of responses to leaseset lookups
- Expand floodfill pool by enabling class 'N' floodfills
- Randomize padding inside encrypted SSU packets
- Preparation for better SSU protocol obfuscation

### Mejoras en el anonimato

- Fix newer lease sets not getting stored or published
- Fix classpath bug when used with 4-year-old installations, causing the console not to start
- Fix addressbook database bug preventing update of the reverse index
- Fix i2psnark bug that changed the infohash of torrents created by Robert and fetched via magnet link
- Fix version checking for plugins
- Fix a streaming timer bug causing frequent IRC disconnects (also affects other close-on-idle tunnels)

### Correcciones de errores

- Don't install as a service on Windows by default
- Reduce transport idle timeouts
- Reduce tunnels on idle in i2psnark
- Change default in i2ptunnel GUI to 3 hops
- IE 10 support
- Individual expiration times in leases, for efficiency on destinations with a high number of tunnels
- Low-level encryption and XOR speedups
- Jetty 7.6.11
- Tomcat 6.0.37
- Translation updates: Chinese, French, German, Portuguese, Russian, Spanish
- New Turkish translation
- Wrapper 3.5.19 (new installs and PPA only)
- Update GeoIP data (new installs and PPA only)
