## Detalles de la actualización

La 0.9.49 continúa el trabajo para hacer que I2P sea más rápido y más seguro. Tenemos varias mejoras y correcciones para el transporte SSU (UDP) que deberían traducirse en mayor velocidad. Esta versión también inicia la migración hacia un nuevo y más rápido cifrado ECIES-X25519 para routers. (Los destinos han estado usando este cifrado desde hace algunas versiones) Hemos estado trabajando en las especificaciones y los protocolos para el nuevo cifrado desde hace varios años, y ¡estamos cerca de concluirlo! La migración requerirá varias versiones para completarse.

Para esta versión, para minimizar las interrupciones, solo las instalaciones nuevas y un porcentaje muy pequeño de las instalaciones existentes (seleccionadas aleatoriamente al reiniciar) utilizarán el nuevo cifrado. Si tu router hace "rekey" (cambio de claves) para usar el nuevo cifrado, puede que tenga menos tráfico o menor fiabilidad de lo habitual durante varios días después de que reinicies. Esto es normal, porque tu router ha generado una nueva identidad. El rendimiento debería recuperarse al cabo de un tiempo.

Hemos "rotado las claves" de la red dos veces antes, al cambiar el tipo de firma predeterminado, pero esta es la primera vez que cambiamos el tipo de cifrado predeterminado. Esperamos que todo transcurra sin problemas, pero vamos a empezar lentamente para asegurarnos.

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## DETALLES DE LA VERSIÓN

**Cambios**

- Build: Git migration
- Build: Move web resources to wars
- i2psnark WebSeed support
- i2psnark padding file support
- i2ptunnel: Move proxy resources to jar
- Router: Redesign ECIES encryption for floodfills (proposal 156)
- Router: Verify RI stores after startup
- Router: Reduce Sybil threshold
- Router: ECIES for new routers
- Router: Start of ECIES migration
- SSU: Send individual fragments of messages
- SSU: Westwood+ congestion control
- SSU: Fast retransmit

**Correcciones de errores**

- Build: Fix Gradle build
- Crypto: Increase ratchet tag window to prevent message loss
- I2CP: Fix encrypted leaseset combined with ECIES crypto or offline keys
- i2ptunnel: Fix config file saving issues
- Router: Fix leaseset request fails causing watchdog to bark
- Router: Hidden mode fixes
- SSU: Fix partial acks not being sent
- SSU: Fix occasional high CPU usage

**Otros**

- Crypto: AES performance improvements
- DoH: Change to RFC 8484 style
- i2ptunnel: Remove DSA shared clients
- Proxy: Add jump servers
- Router: Add more countries for hidden mode
- Router: Tunnel peer selection changes
- Router: Move Sybil subsystem from console to router for embedded use
- Router: Verify RI stores for a while after startup
- Util: New unit tests
- Translation updates
