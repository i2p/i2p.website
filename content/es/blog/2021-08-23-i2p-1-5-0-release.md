## Detalles de la actualización

Sí, así es: después de 9 años de versiones 0.9.x, vamos directamente de la 0.9.50 a la 1.5.0. Esto no significa un cambio importante en la API ni implica que el desarrollo esté terminado. Es simplemente un reconocimiento de casi 20 años de trabajo para brindar anonimato y seguridad a nuestros usuarios.

Esta versión finaliza la implementación de mensajes de construcción de tunnel más pequeños para reducir el ancho de banda. Seguimos con la transición de los routers de la red al cifrado X25519. Por supuesto, también hay numerosas correcciones de errores y mejoras de rendimiento.

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## DETALLES DE LA VERSIÓN

**Cambios**

- RRD4J 3.8
- Tunnels: Finish support for new build messages (proposal 157)
- Updates: Support for .dmg and .exe updates

**Correcciones de errores**

- Console: Fix generation of SSL keys on Java 17
- i2psnark: Fix autostart for magnets
- Router: Fix rare deadlock in publishing our RI
- SSU: Fix handling of bad peer test responses
- UPnP: IPv6 fixes

**Otros**

- Jetty: Improve sort in directory listings
- Jetty: Add X-I2P-Location header
- Router: Increase probability to rekey to ECIES
- Streaming: Performance improvements for low-latency connections
- Translation updates

[Lista completa de errores corregidos](http://trac.i2p2.i2p/query?resolution=fixed&milestone=1.5.0)
