La versión 0.8.3 contiene varias mejoras de rendimiento, incluida la reducción de hilos y del uso de memoria, y una comunicación I2CP (cliente-router) más rápida.

También hay nuevo soporte para la consola del router con SSL, una nueva página de configuración de reseed (resembrado) que incluye soporte para proxy HTTP para quienes estén detrás de cortafuegos restrictivos, una nueva sección de configuración de I2CP que incluye soporte de I2CP-over-SSL para clientes remotos, una nueva sección de configuración de límites de conexiones del servidor y de lista negra para mejorar la protección contra DoS, y una nueva sección de configuración del servidor de salto del proxy HTTP para que pueda añadir fácilmente servidores de salto alternativos. Las estadísticas ahora están limitadas de forma predeterminada para reducir el uso de memoria; el conjunto completo de estadísticas puede volver a habilitarse en la página de configuración de estadísticas. También hay correcciones de errores, por supuesto, así que, como de costumbre, se recomienda actualizar.

Por favor, ofrécete como voluntario en IRC #i2p.

**Detalles de la versión:**

## Nuevas características

* SSL Console
* SSL I2CP
* Jump list, blacklist, and connection limits now on i2ptunnel edit forms
* Smarter update handler that tries multiple sources
* New reseed configuration page for those behind restrictive firewalls
* New I2CP configuration form

## Reducción de recursos

* More efficient I2CP for internal clients
* Less threads, better thread pooling
* Adjust thread count based on available memory
* Change full stats default to false
* Caching of several data structures
* Naming lookup support over existing I2CP sessions

## Correcciones de errores

* Fix i2psnark buttons in Firefox 4.0b and IE
* Restore library functions required by Syndie
