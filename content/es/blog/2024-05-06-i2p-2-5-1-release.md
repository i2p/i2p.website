I2P 2.5.1 se publica para abordar ataques de denegación de servicio que afectan a la red y los servicios de I2P. Con esta versión desactivamos las partes basadas en IP de la herramienta de detección de ataques Sybil, que fueron objeto de ataques dirigidos para amplificar el efecto y la duración del ataque. Esto debería ayudar a que la red vuelva a su funcionamiento normal. Quienes hayan desactivado la herramienta de detección de ataques Sybil pueden volver a habilitarla con seguridad. También se han realizado ajustes en otros subsistemas para mejorar la validación de RouterInfo y la selección de pares.

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## Detalles de la versión

### Changes

- Susimail: Add search box
- Susimail: UI Improvements
- NetDB: Don't lookup RI if on banlist
- Tomcat: update to 9.0.88

### Cambios

- Sybil: Disable IP-Closeness Checks in Sybil Attack Analysis Tool
- Profiles: Don't update last heard from if tunnel fails
- NetDB: Improve validation of RI's before storing, sending RI's
