La versión 0.8.13 incluye varias correcciones de errores y un par de funciones nuevas. Nos complace que la última versión haya mejorado significativamente el rendimiento, y que la red esté funcionando bien a pesar del rápido crecimiento continuo. Se recomienda actualizar.

Los archivos están disponibles en la [página de descargas](/downloads).

## DETALLES DE LA VERSIÓN

Es probable que esta sea la última versión antes de 0.9, en la cual migraremos a Jetty 6 e introduciremos una página de inicio simplificada de la consola del router. Consulta la sección de noticias de tu consola del router en las próximas semanas para más información sobre la próxima versión.

A partir de esta versión, el router comprobará e instalará actualizaciones de complementos poco después de reiniciarse tras una actualización del router. Para evitarlo, añade la configuración avanzada `plugins.autoUpdate=false` antes de reiniciar. También hay un nuevo botón de actualizar todo en la página de configuración del cliente.

Routers en ciertos países ahora activarán automáticamente el modo oculto para una mayor protección. Sin embargo, el modo oculto puede tener un rendimiento o una fiabilidad inferiores y aún está en desarrollo. Para desactivar el modo oculto antes de reiniciar, agrega la configuración avanzada `router.isHidden=false`. Para desactivar el modo oculto más tarde, selecciona *use all auto-detect methods* en la sección *IP Configuration* de la página de configuración de red. Para la lista de países, consulta el hilo en zzz.i2p. El único país de la lista que tiene más de uno o dos usuarios de I2P es China.

## Cambios importantes

- Check for and download plugin updates upon restarting after a router update
- Routers in certain countries will now automatically enable hidden mode for increased protection
- New Tunnel Wizard for creating tunnels
- A SIGHUP to the wrapper will now initiate a graceful shutdown

## Correcciones de errores

- Increase read timeout in HTTP Proxy
- Shutdown hooks will now run when router is shutdown externally (i2prouter stop), broken in 0.8.8
- If an external IP address is specified, bind only to that interface
- Fail tunnels we created when we can't contact the adjacent hop
- Prevent races when saving configuration
- For plugins, check min and max Jetty versions; check all version requirements at startup, not just at install
- Fix plugin startup when console is not on port 7657
- Only stop plugins if they are running
- Fix NPE when no network interfaces are present
- Fix eepget exit code on failure
- Improve inbound tunnel building when in hidden mode
- Publish our router info sooner after startup to facilitate inbound tunnel building
- Fix Streamr tunnel registration

## Otros

- Add icons to buttons in SusMail and SusiDNS
- Move wrapper PID, status, and log files from /tmp to ~/.i2p (Debian packages and new installs only)
- i2prouter graceful (Debian packages and new installs only)
- Increase number of floodfills
- Repack jars in installer to save a little space
- New translation infrastructure for i2prouter script (not enabled yet)
- New Czech translation (thanks Waseihou)
- German, Italian, Polish, Spanish, Swedish, Ukrainian translation updates
