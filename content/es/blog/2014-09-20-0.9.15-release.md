0.9.15 incorpora compatibilidad preliminar con firmas EdDSA Ed25519. Incluye un nuevo backend de configuración persistente para i2psnark y corrige varios problemas con el manejo de nombres de archivo por parte de i2psnark. Hay varias mejoras para acelerar SAM. Los complementos ahora admiten firmas más robustas en el formato de archivo su3. La instalación de complementos a través de la consola, que se había deshabilitado en la 0.9.14, vuelve a estar habilitada.

Hemos admitido firmas ECDSA desde la versión 0.9.12, y nos gustaría empezar a utilizar ECDSA de forma predeterminada. Desafortunadamente, algunos de ustedes aún están ejecutando versiones anteriores de I2P y, en otros casos, su distribución o el entorno de ejecución de Java no admite ECDSA. Se informa que las distribuciones de Red Hat (RHEL, Fedora) no incluyen compatibilidad con ECDSA. Algunos han solucionado los problemas de Java actualizando de Java 6 a Java 7; otros han tenido éxito instalando los "archivos de políticas de fuerza ilimitada". Hemos añadido información sobre la criptografía ausente al archivo de registro y a la página /logs en la consola. Después de actualizar a la versión 0.9.15, por favor compruebe si le falta compatibilidad con ECDSA e intente solucionarlo si es necesario. Esto es particularmente importante para quienes operan eepsites y servicios populares.

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## DETALLES DE LA VERSIÓN

### Changes

- Add support for Ed25519 signatures
- i2psnark move to separate config file for each torrent to better support per-torrent settings
- Add i2psnark support for data outside the i2psnark/ directory
- Enable stronger signatures (su3 format) for plugins
- Speed up SSU introductions by responding to hole punch messages
- Several improvements in SAM efficiency

### Cambios

- Form submission fixes in the console and i2psnark
- Streaming fixes for long signatures
- i2psnark fixes for file name character mapping when seeding
- I2PTunnel fixes stopping client tunnels
- I2PTunnel fix updating options on a running delay-open client tunnel

### Correcciones de errores

- Re-enable plugin installation via the console, removed in 0.9.14
- i2psnark now remembers uploaded count across restarts
- i2psnark increase max piece size to 8 MB
- i2psnark several UI fixes
- Prohibit SSU peer test requests unless a connection is established
- i2ptunnel add support for local SSL connections for standard and IRC client tunnels
- Console and log warnings for unavailable crypto
- More consistent routing for Delivery Status Messages to reduce network connections
- Disable external entities in UPnP XML parser
- More escaping and cleanups in forms and messages
- Translation updates
- Update GeoIP data (in both new installs and updates)
