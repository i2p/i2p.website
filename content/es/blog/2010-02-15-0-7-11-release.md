La versión 0.7.11 corrige varios problemas, algunos relacionados con los relojes, que impedían que los nuevos routers se integraran rápidamente en la red. También corrige el instalador para Windows de 64 bits. La versión incluye una actualización a [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) para corregir [esta vulnerabilidad](http://www.kb.cert.org/vuls/id/402580).

Como de costumbre, esta versión incluye más mejoras de anonimato y rendimiento, y una variedad de correcciones de errores. Se recomienda actualizar.

¡Por favor, envía a los desarrolladores tus comentarios en IRC #i2p o en [forum.i2p2.de](http://forum.i2p2.de/), [participa](http://www.i2p2.de/getinvolved.html), corre la voz y [dona](http://www.i2p2.de/donate.html)! Todavía estamos buscando ayuda para traducciones nuevas y existentes. Por favor, ofrécete como voluntario en IRC #i2p.

Los archivos están disponibles en la [página de descargas](/downloads/).

Detalles de la versión:

## Instalador

- Fix installations on 64-bit Windows by upgrading to launch4j 3.0.1 and adding 64-bit dll to izpack.
- The wrapper is still 32-bit only. 64-bit Windows users should start I2P with the 'No Window' icon if the 'Restartable' icon does not work.

## Transporte

- Better handle clock skews at startup, even if NTP does not work because UDP is blocked or other reasons
- Fix clock skew reporting
- Bootstrap profiles when loading the netDb from disk (broken in 0.7.8)

## Servidor web integrado (Jetty)

- Upgrade to [Jetty 5.1.15](http://old.nabble.com/jetty-5.1.15,-6.1.18-and-7.0.0.M2-td23652819.html) which fixes [this vulnerability](http://www.kb.cert.org/vuls/id/402580).
- Turn on checkAliases for increased protection

## Base de datos de la red

- Add some DOS protection for floodfills
- Increase the number of floodfills again

## Otros

- Faster startup
- Bug fixes in i2psnark, BOB, clock handling, and more
