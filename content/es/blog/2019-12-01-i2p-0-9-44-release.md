## Detalles de la actualización

0.9.44 contiene una corrección importante para un problema de denegación de servicio en los servicios ocultos al manejar nuevos tipos de cifrado. Todos los usuarios deben actualizar lo antes posible.

Esta versión incluye soporte inicial para un nuevo cifrado de extremo a extremo (propuesta 144). El trabajo en este proyecto continúa y todavía no está listo para su uso. Hay cambios en la página de inicio de la consola y nuevos reproductores de medios HTML5 integrados en i2psnark. Se incluyen correcciones adicionales para redes IPv6 detrás de cortafuegos. Las correcciones en la construcción de Tunnel (túneles de I2P) deberían resultar en un inicio más rápido para algunos usuarios.

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## DETALLES DE LA VERSIÓN

**Cambios**

- Console home page changes
- Initial code for new encryption (proposal 144)
- Router changes to support multiple encryption types
- HTML5 players in i2psnark
- Audio playlist in i2psnark
- Change Windows data location to %LOCALAPPDIR% (new installs only)

**Correcciones de errores**

- Fix AppArmor config for Java 11
- Fix handling of unsupported encryption types
- IPv6 firewalled fixes
- Fix long stop delay for some plugins
- Fix custom icons for some plugins
- Fix exploratory tunnel build issues causing long startup times
- Prevent multiple zero-hop tunnels in a pool
- Recognize gettext 0.20 in build scripts

**Otros**

- Don't check clove ID in Bloom filter
- Reduce SessionTag memory usage
- Change default sig type to EdDSA for Android
- Translation updates
- Update GeoIP data
