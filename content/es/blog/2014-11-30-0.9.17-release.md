0.9.17 es principalmente una versión de corrección de errores, pero también continúa nuestra migración hacia firmas criptográficas más sólidas.

Hemos migrado el sistema de fuentes de noticias que se utiliza para las noticias en su consola y la indicación de la versión más reciente del router a un formato firmado mediante claves RSA de 4096 bits para mayor seguridad.

Nuevos eepsites y servidores se firmarán con ECDSA de forma predeterminada, si ECDSA está disponible. Ahora hay una advertencia en la barra lateral de la consola si ECDSA no está disponible. Para los usuarios de RedHat, tenemos informes de instalaciones exitosas del jar BouncyCastle Provider (bcprov) para añadir compatibilidad con ECDSA.

Hemos corregido varios errores graves, incluido un problema de corrupción de paquetes SSU y un error de SAM que afecta a i2p-messenger y a otras aplicaciones SAM. Hay varias correcciones para las firmas ECDSA preliminares del router añadidas en la última versión, pero aún no activadas.

Muchos de nosotros asistiremos al 31C3 en Hamburgo en diciembre. ¡Pásate por nuestra mesa y salúdanos!

Como de costumbre, recomendamos actualizar a esta versión. La mejor manera de mantener la seguridad y ayudar a la red es ejecutar la versión más reciente.

## DETALLES DE LA VERSIÓN

### Changes

- Signed news
- ECDSA default for new server tunnels
- Reseeding now SSL-only by default

### Cambios

- Fix SSU sending corrupt ack-only packets with partial bitfields
- Fix SSU inbound connection fail from non-DSA router
- Don't select incompatible peers if we are a non-DSA router
- Fix EdDSA signature verification bug
- Set I2NP lookup type flags in all cases, not just when a reply tunnel is used
- Stop i2ptunnel server acceptor thread after close
- Fix bug preventing some plugins from stopping completely
- Fix SAM v3 bug causing failures in incoming connections

### Correcciones de errores

- Add a warning in the console sidebar if ECDSA not supported
- Log warnings for Java 6 that we will eventually require Java 7
- Don't let proxied routers auto-floodfill
- Don't resend SSU acks that are too old
- Don't publish direct info in SSU address if introducers are required
- New default opentrackers in i2psnark
- Add support for specifiying data directory per-torrent in i2psnark
- Changes in streaming accept() error behavior
- Minor blockfile format changes
- New option for persistent random key to preserve peer ordering across restarts
- Translation updates
- Update GeoIP data (new installs and PPA only)
