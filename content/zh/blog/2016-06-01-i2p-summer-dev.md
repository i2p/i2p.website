在过去几年里，用户掌控自身数据的必要性变得愈发明显。随着 Signal 等消息应用以及 Tahoe-LAFS 等文件存储系统的兴起，这方面已经取得了出色的进展。Let's Encrypt 为将 HTTPS 普及到全球所做的持续努力也在稳步取得进展。

But building privacy and anonymity into applications is not trivial. Much of the software people use every day was not designed to be privacy-preserving, and the tools developers have available are generally not easy to work with. The recently-published OnionScan survey gives some insight into just how easy it is for even technical users to mis-configure their services, completely undermining their intentions.

## 帮助开发者帮助他们的用户

我们很高兴地宣布，今年夏天，I2P 将启动一项旨在改进隐私软件生态系统的开发计划。我们的目标是，让希望在其应用程序中利用 I2P 的开发者，以及尝试通过 I2P 配置并运行其应用程序的用户，都能更轻松。

我们今年夏天将把时间重点投入到三个相辅相成的领域：

### June: APIs

在六月，我们将更新用于与 I2P 对接的各种库。今年，我们在为 SAM API 扩展附加功能方面取得了重大进展，例如对数据报和端口的支持。我们计划在我们的 C++ 和 Python 库中让这些功能易于使用。

我们也将很快让 Java 和 Android 开发者更容易在他们的应用中添加 I2P 支持。敬请期待！

### 六月：API

七月，我们将与那些表示有意增加对 I2P 支持的应用合作。当前隐私领域正在涌现许多非常出色的理念，我们希望帮助这些社区充分利用十余年来在点对点匿名性方面的研究与开发成果。让这些应用能够在 I2P 上原生运行是提升可用性的重要一步，同时也会改进它们对用户信息的考量与处理方式。

### 七月：应用

最后，在八月我们将把注意力转向我们在 I2P 中随附的应用，以及更广泛的各类插件。其中有些确实需要好好打磨一下，使其更易用，同时修复尚存的缺陷！我们希望 I2P 的长期支持者会喜欢这项工作的成果。

## Take part in Summer Dev!

在这些领域，我们还有许多希望完成的想法和计划。如果你对动手开发隐私与匿名软件、设计易用的网站或界面、或为用户撰写指南感兴趣，欢迎在 IRC 或 Twitter 上与我们交流！我们始终乐于欢迎新成员加入我们的社区。我们会向所有参与的新贡献者寄送 I2P 贴纸！

同样，如果你是希望将 I2P 集成到应用中的开发者，需要一些帮助，或者只是想聊聊相关概念或细节：欢迎联系！
如果你想参与我们的七月应用月活动，可在 Twitter 上联系 @GetI2P、@i2p 或 @str4d。
你也可以在 OFTC 或 FreeNode 的 #i2p-dev 找到我们。

我们会随着进展在这里发布更新，你也可以在 Twitter 上使用标签 #I2PSummer 关注我们的进展，并分享你自己的想法和作品。迎接这个夏天吧！
